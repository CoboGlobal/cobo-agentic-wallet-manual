## Business Context

Your DCA agent should buy continuously, but not on the wrong chain or outside risk limits. You need automated guardrails that deny invalid routes and let the agent recover with compliant parameters.

Core flow:

- Agent submits a pact requesting ETH-only transfer rights within spend limits
- Owner approves in the Web Console
- Agent executes compliant ETH buys within the approved boundaries
- Policy denies any attempt that exceeds limits; agent retries with a compliant amount

<Accordion title="Policy Architecture">
```text
Owner (human)
  -> Delegation
       - Operator: trader principal
       - Permissions: [write:transfer]
       - Policies:
           - allowed_chains: [ETH]
           - max_per_tx: 10
           - daily_cumulative: 25
           - monthly_cumulative: 200
```
</Accordion>

## Pact Specification

The agent submits this PactSpec when requesting authorization. The owner reviews the chain restriction and cumulative spend limits before approving.

```json
{
  "permissions": ["write:transfer", "read:wallet"],
  "policies": [
    {
      "name": "allow-eth-only",
      "type": "transfer",
      "rules": {
        "effect": "allow",
        "when": {
          "chain_in": ["SETH"]
        }
      }
    },
    {
      "name": "deny-over-limits",
      "type": "transfer",
      "rules": {
        "effect": "deny",
        "when": {},
        "deny_if": {
          "amount_usd_gt": "10",
          "usage_limits": {
            "rolling_24h": { "amount_usd_gt": "25" },
            "rolling_30d": { "amount_usd_gt": "200" }
          }
        }
      }
    }
  ],
  "duration_seconds": 7776000
}
```

## Full Flow

### Step 1: Owner sets the task

The owner gives the agent a natural-language instruction:

> "Run my DCA strategy. ETH only, max $10 per buy, $25 daily, $200 monthly. Valid 90 days."

### Step 2: Agent submits authorization request

The agent parses the instruction and submits a pact for owner approval.

import asyncio, os
from cobo_agentic_wallet import WalletAPIClient

async def main():
    async with WalletAPIClient(api_key=os.environ["AGENT_WALLET_API_KEY"]) as client:
        wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]

        # Submit pact
        result = await client.submit_pact(
            wallet_id=wallet_id,
            intent="Run DCA strategy: ETH only, max $10/buy, $25 daily, $200 monthly, valid 90 days",
            spec={
                "permissions": ["write:transfer", "read:wallet"],
                "policies": [
                    {
                        "name": "allow-eth-only",
                        "type": "transfer",
                        "rules": {
                            "effect": "allow",
                            "when": {"chain_in": ["SETH"]},
                        },
                    },
                    {
                        "name": "deny-over-limits",
                        "type": "transfer",
                        "rules": {
                            "effect": "deny",
                            "when": {},
                            "deny_if": {
                                "amount_usd_gt": "10",
                                "usage_limits": {
                                    "rolling_24h": {"amount_usd_gt": "25"},
                                    "rolling_30d":  {"amount_usd_gt": "200"},
                                },
                            },
                        },
                    },
                ],
                "duration_seconds": 7776000,
            },
        )
        pact_id = result["pact_id"]
        print(f"Pact {pact_id} submitted — awaiting owner approval in Web Console")

        # Poll for activation
        while True:
            pact = await client.get_pact(pact_id)
            if pact["status"] == "active":
                break
            await asyncio.sleep(10)

        print(f"Pact active. Scoped API key: {pact['api_key']}")

asyncio.run(main())

  spec: {

    policies: [
      {

        type: 'transfer',
        rules: {
          effect: 'allow',
          when: { chain_in: ['SETH'] },
        },
      },
      {
        name: 'deny-over-limits',
        type: 'transfer',
        rules: {
          effect: 'deny',
          when: {},
          deny_if: {
            amount_usd_gt: '10',
            usage_limits: {
              rolling_24h: { amount_usd_gt: '25' },
              rolling_30d:  { amount_usd_gt: '200' },
            },
          },
        },
      },
    ],
    duration_seconds: 7776000,
  },
});
console.log(`Pact ${result.pact_id} submitted — awaiting owner approval`);

console.log(`Pact active. Scoped API key: ${pact.api_key}`);

    )
})

### Step 3: Owner approves in Web Console

The pact appears in the Web Console with the intent, execution plan, chain restriction, and cumulative spend limits. The owner reviews and approves. The agent receives a pact-scoped API key and can begin operating.

### Step 4: Agent operates

With an active pact, the agent executes DCA purchases within the approved limits.
