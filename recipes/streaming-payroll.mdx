## Business Context

An agent manages salary streams for a team using Superfluid on Base Sepolia (testnet). It creates payment streams for new employees, adjusts rates when compensation changes, and terminates streams when an employee leaves. Salaries flow continuously per second — no monthly batch runs, no delayed payouts.

<Accordion title="Permission Architecture">
```text
Owner (human)
  -> Pact (90 days)
       - Operator: agent principal
       - Permissions: [write:contract_call, read:wallet]
       - Policies:
           - allowed_contracts: [SUPERFLUID_CFAV1_FORWARDER]
           - chain: TBASE_SETH (Base Sepolia)
```
</Accordion>

## Pact Specification

```json
{
  "permissions": ["write:contract_call", "read:wallet"],
  "policies": [
    {
      "name": "allow-superfluid",
      "type": "contract_call",
      "rules": {
        "effect": "allow",
        "when": {
          "chain_in": ["TBASE_SETH"],
          "target_in": [
            {
              "chain_id": "TBASE_SETH",
              "contract_addr": "<SUPERFLUID_CFAV1_FORWARDER>"
            }
          ]
        }
      }
    }
  ],
  "duration_seconds": 7776000
}
```

<Info>
This recipe targets Base Sepolia (`TBASE_SETH`) for testnet use. To run on Base mainnet, change `chain_in` and `target_in` to `BASE_ETH` and use the Base mainnet Superfluid CFAv1 Forwarder address.
</Info>

## Step-by-Step

### Step 1: Owner sets the task

> **Owner → Agent:** "Set up streaming salaries for our team. Alice gets $5,000/month, Bob gets $3,000/month. Manage these streams — create, adjust, and stop them as needed — for the next 90 days."

### Step 3: Owner approves in Web Console

The agent notifies the owner that an authorization request is pending. The owner reviews the allowed contract (Superfluid CFAv1 Forwarder on Base Sepolia) and the 90-day duration, then approves.

### Step 4: Agent creates streams

Superfluid streams are defined by a flow rate in tokens per second (in the token's native decimals). The CFAv1 Forwarder's `createFlow` function takes the super token address, receiver, and flow rate.

<Tabs>
  <Tab title="CLI">
```bash
# USDC on Base Sepolia has 6 decimals.
# $5,000/month = 5000 * 10^6 / (30 * 24 * 3600) = 1929 wei/s (approx)
ALICE_FLOW_RATE=1929012345  # tokens per second in micro-USDC

caw tx call "$WALLET_UUID" \
  --chain TBASE_SETH \
  --to "<SUPERFLUID_CFAV1_FORWARDER>" \
  --calldata "$(cast calldata 'createFlow(address,address,address,int96,bytes)' '<SETH_USDC_ADDRESS>' '<WALLET_ADDR>' '<ALICE_ADDRESS>' $ALICE_FLOW_RATE '0x')"
```

```python
import os
from cobo_agentic_wallet import WalletAPIClient

# Calldata encoders — replace with real ABI encoding (e.g., web3.py or eth_abi)
def encode_create_flow(token: str, sender: str, receiver: str, flow_rate: int) -> str: ...
def encode_update_flow(token: str, sender: str, receiver: str, flow_rate: int) -> str: ...
def encode_delete_flow(token: str, sender: str, receiver: str) -> str: ...

MONTH_SECONDS = 30 * 24 * 3600
USDC_DECIMALS = 10 ** 6  # USDC has 6 decimals

def monthly_to_flow_rate(monthly_usd: float) -> int:
    """Convert a monthly USD salary to a Superfluid flow rate (tokens/second)."""
    return int(monthly_usd * USDC_DECIMALS / MONTH_SECONDS)

async def create_streams(pact: dict):
    wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]
    pact_client = WalletAPIClient(api_key=pact["api_key"])

    team = [
        {"name": "Alice", "address": "<ALICE_ADDRESS>", "monthly_usd": 5000},
        {"name": "Bob",   "address": "<BOB_ADDRESS>",   "monthly_usd": 3000},
    ]

    for member in team:
        flow_rate = monthly_to_flow_rate(member["monthly_usd"])
        await pact_client.contract_call(
            wallet_id=wallet_id,
            chain_id="TBASE_SETH",
            contract_addr="<SUPERFLUID_CFAV1_FORWARDER>",
            calldata=encode_create_flow(
                token="<SETH_USDC_ADDRESS>",
                sender=wallet_id,
                receiver=member["address"],
                flow_rate=flow_rate,
            ),
        )
        print(
            f"Stream created for {member['name']}: "
            f"${member['monthly_usd']:,}/month ({flow_rate} tokens/s)"
        )
```

```typescript
import { WalletAPIClient } from 'cobo-agentic-wallet';
import { encodeFunctionData } from 'viem';

const CFA_FORWARDER_ABI = [...]; // Superfluid CFAv1Forwarder ABI

const MONTH_SECONDS = 30 * 24 * 3600;
const USDC_DECIMALS = 1_000_000n; // 6 decimals

function monthlyToFlowRate(monthlyUsd: number): bigint {
  return BigInt(Math.floor(monthlyUsd * 1_000_000)) / BigInt(MONTH_SECONDS);
}

async function createStreams(pact: { api_key: string }) {
  const walletId = process.env.AGENT_WALLET_WALLET_ID!;
  const pactClient = new WalletAPIClient({ apiKey: pact.api_key });

  const team = [
    { name: 'Alice', address: '<ALICE_ADDRESS>', monthlyUsd: 5000 },
    { name: 'Bob',   address: '<BOB_ADDRESS>',   monthlyUsd: 3000 },
  ];

  for (const member of team) {
    const flowRate = monthlyToFlowRate(member.monthlyUsd);
    const calldata = encodeFunctionData({
      abi: CFA_FORWARDER_ABI,
      functionName: 'createFlow',
      args: ['<SETH_USDC_ADDRESS>', walletId, member.address, flowRate, '0x'],
    });
    await pactClient.contractCall({
      wallet_id: walletId,
      chain_id: 'TBASE_SETH',
      contract_addr: '<SUPERFLUID_CFAV1_FORWARDER>',
      calldata,
    });
    console.log(`Stream created for ${member.name}: $${member.monthlyUsd.toLocaleString()}/month`);
  }
}
```

### Step 5: Adjust or stop a stream

When compensation changes or an employee leaves, the agent updates or deletes the stream.

<Tabs>
  <Tab title="CLI">
```bash
# Adjust Alice's salary to $6,000/month
NEW_FLOW_RATE=2314814814  # 6000 * 10^6 / (30 * 24 * 3600)

caw tx call "$WALLET_UUID" \
  --chain TBASE_SETH \
  --to "<SUPERFLUID_CFAV1_FORWARDER>" \
  --calldata "$(cast calldata 'updateFlow(address,address,address,int96,bytes)' '<SETH_USDC_ADDRESS>' '<WALLET_ADDR>' '<ALICE_ADDRESS>' $NEW_FLOW_RATE '0x')"

# Terminate Bob's stream
caw tx call "$WALLET_UUID" \
  --chain TBASE_SETH \
  --to "<SUPERFLUID_CFAV1_FORWARDER>" \
  --calldata "$(cast calldata 'deleteFlow(address,address,address,bytes)' '<SETH_USDC_ADDRESS>' '<WALLET_ADDR>' '<BOB_ADDRESS>' '0x')"
```

<Tip>
Superfluid streams require the sender's wallet to maintain a sufficient token buffer (typically 4 hours of flow rate). Monitor the wallet balance and top it up before it runs dry, or all active streams will be liquidated.
</Tip>

## Audit

```bash
caw audit logs --wallet "$WALLET_UUID" --action contract_call --format table
```

Each stream operation (create, update, delete) appears as a separate contract call entry. The audit log lets you verify which streams were opened, when rates were adjusted, and when streams were terminated.
