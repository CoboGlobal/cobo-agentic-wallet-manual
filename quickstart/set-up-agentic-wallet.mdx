This guide walks you through creating a wallet and connecting your AI agent to it. By the end, your agent will have a live wallet it can use to transfer tokens and call contracts, operating within the spending limits and approval rules you set.

## Before you start

Choose your setup mode based on who should hold the signing keys and how much oversight you need.

| | Supervised | On-device | Autonomous |
|---|---|---|---|
| **Key shares on** | Agent's machine | Your phone | Agent's machine |
| **Transaction signing** | Automatic | Manual — you approve each transaction | Automatic |
| **Wallet owned by** | You | You | Agent (can claim later) |
| **Best for** | Automated operations with owner oversight | Large funds or high security | Fully autonomous agents, prototypes |

Jump to your setup mode: [Supervised](#supervised) · [On-device](#on-device) · [Autonomous](#autonomous)

## Supervised

<img src="/images/onboarding-supervised.svg" alt="Supervised onboarding: sign in to Web Console, run setup instruction, agent completes 5 steps, set policies" />

The most common setup. You generate a pairing code in the Web Console and give it to your agent. The agent registers its identity, generates key shares, and creates the wallet — all in one command. You can configure policies in the Web Console while the agent sets up; they are applied automatically once the wallet is ready.

### 1. Sign up and generate a pairing code

Go to the Web Console and sign in. Click **Connect Agent** to generate a pairing code and a ready-made setup instruction.

### 2. Run the setup instruction in your agent

Copy the instruction and run it in your agent. Under the hood it runs:

```bash
caw onboard --token <PAIRING_CODE> --create-wallet
```

The agent completes 5 steps (60–180 seconds total):

| Step | What happens |
|---|---|
| **[1/5] Register identity** | Uses your pairing code to register with Cobo and get an API key |
| **[2/5] Download TSS node** | Downloads the signing node binary (~20 MB) |
| **[3/5] Initialize TSS node** | Generates local key share configuration |
| **[4/5] Start TSS node** | Starts the signing node process |
| **[5/5] Create wallet** | Submits to Cobo, which creates the wallet and runs key generation. Wallet becomes active once complete. |

### 3. Set policies (can do while agent is setting up)

You don't need to wait. In the Web Console, set your spending limits, allowed networks, and approval thresholds. They are applied to the wallet automatically once it's active.

### 4. Create an address and claim testnet funds

Once your agent reports the wallet is active:

```bash
caw wallet address create --chain SETH
caw faucet claim --coin SETH
```

## On-device

<img src="/images/onboarding-on-device.svg" alt="On-device onboarding: create wallet in Human App, tap Connect Agent, run setup instruction, delegate wallet" />

Your phone holds the key shares. You initiate setup from the Human App and your agent registers its identity only — no wallet creation. The wallet stays on your device; you then delegate it to the agent from the Human App.

### 1. Sign up and create your wallet

Download the Human App and sign in. The app walks you through wallet creation. Key shares are generated on your device and you must approve every transaction manually. The wallet is usable immediately, with or without an agent.

### 2. Tap Connect Agent

In the Human App, tap **Connect Agent** to get a pairing code and a ready-made setup instruction.

### 3. Run the setup instruction in your agent

Copy the instruction and run it in your agent. Under the hood it runs:

```bash
caw onboard --token <PAIRING_CODE>
```

This completes in one step — just identity registration. No wallet is created by the agent.

### 4. Delegate your wallet to the agent

Back in the Human App, tell the chat assistant:

> "I want to authorize my agent to use my wallet"

The assistant walks you through the delegation and confirms when it's done.

### 5. Adjust policies (optional)

You can adjust policies at any time via the Human App chat assistant.

## Autonomous

<img src="/images/onboarding-autonomous.svg" alt="Autonomous onboarding: run onboard command, agent completes 5 steps, optionally claim wallet" />

No human involvement required. The agent registers itself, generates key shares, and creates its own wallet — making the agent the wallet owner. Useful for fully autonomous agents and prototypes. You can claim the wallet later to transfer ownership to yourself.

### 1. Run the onboard command

```bash
caw onboard --create-wallet
```

The agent completes the same 5 steps as Supervised (60–180 seconds), but without a pairing code. The agent is registered as the wallet owner and no delegation is created.

### 2. Create an address and claim testnet funds

```bash
caw wallet address create --chain SETH
caw faucet claim --coin SETH
```

### 3. Claim the wallet (optional)

To transfer wallet ownership to yourself at any time:

```bash
caw profile claim
```

This generates a claim link. Open it in your browser, authenticate with your account, and ownership transfers to you. The agent's access is preserved — it continues operating the wallet under your ownership and policies.

## What your agent can do after setup

Once set up, your agent understands natural language wallet commands:

| You say | What happens |
|---|---|
| "Show my wallet balance" | Runs `caw wallet balance` |
| "Transfer 10 USDC to 0x1234 on Base" | Runs `caw tx transfer` with policy enforcement |
| "Show recent transactions" | Runs `caw tx list` |
| "List wallets I can access" | Runs `caw wallet list` |
| "Check if I can call this contract before executing" | Runs `caw policy dry-run` |

## FAQ

<AccordionGroup>

<Accordion title="My pairing code expired — what do I do?">
Refresh it from wherever you created it (Web Console or Human App). The new code works exactly the same way. Nothing about your account or setup is affected.
</Accordion>

<Accordion title="How long does wallet setup take?">
The 5-step setup (Supervised and Autonomous modes) typically takes 60–180 seconds. Step 2 downloads the TSS node binary (~20 MB) and steps 3–5 run key share generation. As long as the command hasn't returned an error, let it run.
</Accordion>

<Accordion title="What if onboarding fails or times out?">
Your pairing code remains valid for 15 minutes. If a profile was partially created, archive it before retrying:

```bash
caw profile archive
caw onboard --token <PAIRING_CODE> --create-wallet
```

If the pairing code has expired, refresh it from the Web Console or Human App first.
</Accordion>

<Accordion title="What does my agent do with the pairing code?">
The agent uses it to register with Cobo and get an API key tied to your account. The API key is what the agent uses for all subsequent wallet operations — the pairing code is only needed once.
</Accordion>

<Accordion title="Where are credentials stored?">
Each onboarding creates an isolated profile under `~/.cobo-agentic-wallet/profiles/profile_{agent_id}/` containing the API key, wallet UUID, and TSS node files. Use `caw profile list` to see all profiles and `caw profile use <agent_id>` to switch between them.
</Accordion>

<Accordion title="Is wallet data shared across agents using the same API key?">
Yes. All wallet data, policies, and transaction history are shared under a single unified account. Any agent holding the same API key sees the same wallet state and delegation scope.
</Accordion>

<Accordion title="I already onboarded — can I run caw onboard again?">
If your existing API key is still valid, `caw onboard` will refuse to run to prevent accidental overwrites. To provision a new agent, generate a new pairing code from the Web Console or Human App.
</Accordion>

<Accordion title="Can multiple agents operate the same wallet?">
Yes — share the same API key across as many agent programs as you like. They all operate the same wallet under the same delegation and policies.
</Accordion>

<Accordion title="Can my agent operate multiple wallets?">
No. An agent can only have one API key, and one API key can only operate one wallet.
</Accordion>

<Accordion title="Can I create multiple wallets?">
Yes. Generate a new pairing code from the Web Console or Human App and go through the full onboarding process with a new agent.
</Accordion>

<Accordion title="Do I need to prepare gas tokens?">
No. Gas fees are covered automatically via Cobo Gasless — you don't need to fund any gas wallet before transacting.
</Accordion>

</AccordionGroup>
