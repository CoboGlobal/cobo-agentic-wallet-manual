## Why DeFi requires an approve step

Unlike native ETH (which can be sent as part of a transaction's `value` field), ERC-20 tokens sit in a contract's mapping. No one can move them except the owner — or someone the owner has explicitly authorized.

DeFi protocols need to pull tokens from your wallet when you deposit, swap, or repay debt. To do this, they call `transferFrom(you, destination, amount)` on the token contract. But the token contract only allows this if you have previously called `approve(protocol, amount)`.

This is why every DeFi interaction with ERC-20 tokens is a **two-step process**:

1. **Approve** — authorize the protocol to pull up to `amount` tokens from your balance.
2. **Interact** — call the DeFi protocol function (swap, deposit, repay, etc.).

## The approve/allowance model

The ERC-20 standard maintains an `allowance` mapping:

```
allowance[owner][spender] = amount
```

When you call `approve(spender, amount)`, you set this mapping. When the spender calls `transferFrom(from, to, amount)`, the token contract checks the allowance, deducts from it, and moves the tokens.

Allowances persist across transactions. Once approved, the spender can move tokens until the allowance is exhausted — or until you revoke it with `approve(spender, 0)`.

## Checking existing allowances

Before approving, check if a sufficient allowance already exists. This saves a transaction:

```python
from web3 import Web3

w3 = Web3(Web3.HTTPProvider("https://rpc.base.org"))

minimal_abi = [{
    "name": "allowance",
    "type": "function",
    "inputs": [
        {"name": "owner", "type": "address"},
        {"name": "spender", "type": "address"}
    ],
    "outputs": [{"name": "", "type": "uint256"}],
    "stateMutability": "view"
}]

token = w3.eth.contract(address="0xUSDCAddress...", abi=minimal_abi)
current_allowance = token.functions.allowance(wallet_address, spender_address).call()

if current_allowance >= amount_needed:
    print("Sufficient allowance already set — skip approve")
else:
    print(f"Need to approve: current={current_allowance}, needed={amount_needed}")
```

## Exact vs. infinite approvals

**Exact approval** — approve only the amount needed for this transaction:

```python
# approve(spender, exact_amount_needed)
```

If the protocol is exploited or has a bug that allows unlimited `transferFrom`, the attacker can drain at most the approved amount.

**Infinite approval** — approve `2**256 - 1` (max uint256), effectively unlimited:

```python
UNLIMITED = 2**256 - 1
# approve(spender, UNLIMITED)
```

Infinite approvals are gas-efficient — you approve once and never again. But if the protocol is ever compromised, all your tokens in that wallet are exposed.

<Note>
For agent wallets, **exact approvals per transaction are recommended**. The marginal gas cost is low compared to the reduced blast radius. Agents can re-approve before each call as part of the standard two-step flow.
</Note>

## Revoking approvals

To revoke an existing approval, call `approve(spender, 0)`:

```python
from web3 import Web3
from eth_abi import encode

selector = Web3.keccak(text="approve(address,uint256)")[:4].hex()
revoke_calldata = "0x" + selector + encode(
    ["address", "uint256"], [spender_address, 0]
).hex()

await client.call_contract(
    wallet_id=wallet_id,
    chain_id="BASE_ETH",
    contract_addr=token_address,
    calldata=revoke_calldata,
)
```

## The two-step helper pattern

A utility function that combines the approve and protocol call:

```python
from web3 import Web3
from eth_abi import encode
from typing import Optional

async def approve_and_call(
    client,
    wallet_id: str,
    chain_id: str,
    token_addr: str,
    spender_addr: str,
    amount: int,
    contract_addr: str,
    contract_calldata: str,
    skip_approve_if_sufficient: bool = True,
) -> dict:
    """Approve token spending and call a contract in two transactions."""

    # Optionally skip approve if allowance already covers the amount
    if skip_approve_if_sufficient:
        w3 = Web3(Web3.HTTPProvider("https://rpc.base.org"))
        abi = [{"name": "allowance", "type": "function",
                "inputs": [{"name": "owner", "type": "address"},
                           {"name": "spender", "type": "address"}],
                "outputs": [{"name": "", "type": "uint256"}],
                "stateMutability": "view"}]
        token = w3.eth.contract(address=token_addr, abi=abi)
        current = token.functions.allowance(wallet_address, spender_addr).call()
        need_approve = current < amount
    else:
        need_approve = True

    if need_approve:
        selector = Web3.keccak(text="approve(address,uint256)")[:4].hex()
        approve_calldata = "0x" + selector + encode(
            ["address", "uint256"], [spender_addr, amount]
        ).hex()
        await client.call_contract(
            wallet_id=wallet_id,
            chain_id=chain_id,
            contract_addr=token_addr,
            calldata=approve_calldata,
        )

    return await client.call_contract(
        wallet_id=wallet_id,
        chain_id=chain_id,
        contract_addr=contract_addr,
        calldata=contract_calldata,
    )
```

## ERC-20 approvals in Pact policies

For every ERC-20 token your agent needs to approve, include an explicit allow rule in the Pact:

```json
{
  "name": "allow-usdc-approve",
  "type": "contract_call",
  "rules": {
    "effect": "allow",
    "when": {
      "chain_in": ["BASE_ETH"],
      "target_in": [
        {"chain_id": "BASE_ETH", "contract_addr": "0xUSDC..."}
      ],
      "function_in": ["approve(address,uint256)"]
    }
  }
}
```

Without an explicit allow rule for the token contract, the approve step is blocked by policy — and the subsequent DeFi call will also fail.

## ERC-2612 permit (gasless approval alternative)

Some tokens (USDC v2, DAI, WETH on certain chains) support **EIP-2612 permit**: an off-chain signed approval that the spender submits alongside the main transaction. This eliminates the separate approve transaction.

The spender calls `permit(owner, spender, value, deadline, v, r, s)` on the token contract, then immediately calls the protocol function — all in a single transaction.

Permit is not universally supported. Check whether the specific token on the specific chain supports ERC-2612 before building permit-based flows.
