This page summarizes the function signatures most commonly used when building DeFi agents. For contract addresses on each network, always check each protocol's official documentation — addresses differ by chain and are updated when new versions are deployed.

## ERC-20

Every ERC-20 token implements this standard interface:

| Function signature | Type | What it does |
|---|---|---|
| `balanceOf(address)` | Read | Returns the token balance of an address |
| `decimals()` | Read | Number of decimal places (usually 6 for USDC, 18 for ETH-like) |
| `symbol()` | Read | Token symbol (e.g., `USDC`, `WETH`) |
| `allowance(address,address)` | Read | Current approval amount for owner → spender |
| `transfer(address,uint256)` | Write | Transfer tokens to a recipient |
| `approve(address,uint256)` | Write | Authorize a spender to pull tokens |
| `transferFrom(address,address,uint256)` | Write | Move tokens on behalf of an approved owner |

## Uniswap v3

### SwapRouter02 — token swaps

| Function | What it does |
|---|---|
| `exactInputSingle((address,address,uint24,address,uint256,uint256,uint160))` | Swap exact input for output via a single pool |
| `exactInput((bytes,address,uint256,uint256))` | Multi-hop swap: encode path as packed bytes |
| `exactOutputSingle((address,address,uint24,address,uint256,uint256,uint160))` | Swap to get an exact output amount |
| `exactOutput((bytes,address,uint256,uint256))` | Multi-hop swap for exact output |

Struct fields for `exactInputSingle`:
- `tokenIn`, `tokenOut` — ERC-20 contract addresses
- `fee` — pool fee tier: `100` (0.01%), `500` (0.05%), `3000` (0.3%), `10000` (1%)
- `recipient` — address that receives `tokenOut`
- `amountIn` — input token amount (raw integer, not decimal)
- `amountOutMinimum` — minimum acceptable output; set to `0` is unsafe
- `sqrtPriceLimitX96` — price limit; use `0` to disable

**Selector reference:**

| Selector | Signature |
|---|---|
| `0x414bf389` | `exactInputSingle(...)` |
| `0xc04b8d59` | `exactInput(...)` |
| `0xdb3e2198` | `exactOutputSingle(...)` |
| `0xf28c0498` | `exactOutput(...)` |

### NonfungiblePositionManager — LP positions

| Function | What it does |
|---|---|
| `mint((address,address,uint24,int24,int24,uint256,uint256,uint256,uint256,address,uint256))` | Create a new LP position (returns NFT `tokenId`) |
| `increaseLiquidity((uint256,uint256,uint256,uint256,uint256,uint256))` | Add liquidity to an existing position |
| `decreaseLiquidity((uint256,uint128,uint256,uint256,uint256))` | Remove liquidity; amounts go to pending fees |
| `collect((uint256,address,uint128,uint128))` | Collect accumulated fees and removed tokens |
| `burn(uint256)` | Destroy an empty position NFT |

For `collect`, pass `MAX_UINT128` (= `2**128 - 1`) for `amount0Max` and `amount1Max` to collect all available.

## Aave v3

### Pool — lending and borrowing

| Function | What it does |
|---|---|
| `supply(address,uint256,address,uint16)` | Deposit tokens; receive aTokens |
| `borrow(address,uint256,uint256,uint16,address)` | Borrow against supplied collateral |
| `repay(address,uint256,uint256,address)` | Repay a borrow |
| `withdraw(address,uint256,address)` | Redeem aTokens and receive underlying |
| `setUserUseReserveAsCollateral(address,bool)` | Enable or disable an asset as collateral |
| `getUserAccountData(address)` | Read health factor, LTV, borrow capacity |
| `liquidationCall(address,address,address,uint256,bool)` | Liquidate an undercollateralized position |

`getUserAccountData` returns (all uint256):
- `totalCollateralBase` — total collateral in USD (8 decimals)
- `totalDebtBase` — total debt in USD
- `availableBorrowsBase` — remaining borrow capacity
- `currentLiquidationThreshold` — % at which position is liquidatable
- `ltv` — loan-to-value ratio
- `healthFactor` — health factor in 1e18 units (1.0 = 1e18)

**Selector reference:**

| Selector | Signature |
|---|---|
| `0x617ba037` | `supply(address,uint256,address,uint16)` |
| `0xa415bcad` | `borrow(address,uint256,uint256,uint16,address)` |
| `0x573ade81` | `repay(address,uint256,uint256,address)` |
| `0x69328dec` | `withdraw(address,uint256,address)` |
| `0xbf92857c` | `getUserAccountData(address)` |

## Curve Finance

### Pool — stable swaps

Curve pool interfaces vary by pool type (v1, v2, NG), but the most common functions are:

| Function | What it does |
|---|---|
| `exchange(int128 i, int128 j, uint256 dx, uint256 min_dy)` | Swap token at index `i` for token at index `j` |
| `add_liquidity(uint256[N] amounts, uint256 min_mint_amount)` | Add liquidity (N = number of tokens in pool) |
| `remove_liquidity(uint256 lp_amount, uint256[N] min_amounts)` | Remove liquidity proportionally |
| `remove_liquidity_one_coin(uint256 lp_amount, int128 i, uint256 min_amount)` | Remove to a single token |
| `get_dy(int128 i, int128 j, uint256 dx)` | Preview swap output (read call) |
| `calc_withdraw_one_coin(uint256 lp_amount, int128 i)` | Preview single-token withdrawal (read) |

Token indices: 0, 1, 2 correspond to the pool's token array (check the specific pool's ABI).

### Gauge — LP staking for CRV emissions

| Function | What it does |
|---|---|
| `deposit(uint256 value)` | Stake LP tokens to earn CRV |
| `withdraw(uint256 value)` | Unstake LP tokens |
| `claim_rewards()` | Claim accumulated reward tokens |
| `claimable_tokens(address)` | Preview claimable CRV (read) |

## Computing selectors

```python
from web3 import Web3

signatures = [
    "transfer(address,uint256)",
    "approve(address,uint256)",
    "supply(address,uint256,address,uint16)",
    "exactInputSingle((address,address,uint24,address,uint256,uint256,uint160))",
    "exchange(int128,int128,uint256,uint256)",
]

for sig in signatures:
    sel = Web3.keccak(text=sig)[:4].hex()
    print(f"0x{sel}  {sig}")
```
