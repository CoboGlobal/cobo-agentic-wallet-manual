## What is a pact?

A pact is the formalized agreement between an agent and its wallet owner. It captures the agent's intent, the execution plan, the spending rules to enforce, and when the authority expires. Approving a pact creates a scoped delegation and a pact-scoped API key automatically — no manual delegation or policy setup required.

## Owner vs. delegate: how the pact flow differs

The pact mechanism behaves differently depending on whether the agent is the wallet owner or a delegate.

| | Agent as owner (before wallet is paired) | Agent as delegate (after wallet is paired) |
|---|---|---|
| **Pact approval** | Automatic — no human approval required | Human (wallet owner) reviews and approves in the Human App |
| **Owner guardrails** | Not applicable — owner has no spending limits | Approval thresholds and spending controls can pause high-risk operations |
| **Pact approval timeout** | Not applicable | 1 hour — unanswered requests expire |

When the wallet owner pairs the wallet via the Human App, the agent transitions from owner to delegate. From that point on, pacts become the mechanism for defining the agent's authority.

**Owner guardrails:** Once the wallet is paired, the owner can set spending and approval controls in the Human App. These guardrails work alongside the pact and can pause operations that require explicit owner approval.

## PactSpec structure

The PactSpec is the JSON payload your agent submits when requesting a pact. It defines the operating boundaries the owner will review.

| Field | Type | Description |
|---|---|---|
| `intent` | string | Plain-language description of what the agent wants to do |
| `execution_plan` | string (markdown) | Detailed strategy — helps the owner understand *how* the agent intends to work |
| `policies` | InlinePolicy[] | Spending rules: transfer limits, allowlists, rolling budgets, approval thresholds |
| `duration_seconds` | integer | How long the pact remains active after activation. Null = no time limit. |
| `completion_conditions` | CompletionCondition[] | Optional early-completion triggers: `TX_COUNT` (total transactions), `AMOUNT_SPENT_USD` (total USD spent) |

### Example PactSpec

A DCA trader pact that allows weekly $500 ETH purchases on Base for 3 months:

```json
{
  "policies": [
    {
      "name": "allow-uniswap-base",
      "type": "contract_call",
      "rules": {
        "effect": "allow",
        "when": {
          "chain_in": ["BASE_ETH"],
          "target_in": [{
            "chain_id": "BASE_ETH",
            "contract_addr": "0x2626664c2603336E57B271c5C0b26F421741e481"
          }]
        },
        "review_if": {
          "amount_usd_gt": "500"
        }
      }
    },
    {
      "name": "deny-excessive-spending",
      "type": "contract_call",
      "rules": {
        "effect": "deny",
        "when": {
          "chain_in": ["BASE_ETH"]
        },
        "deny_if": {
          "amount_usd_gt": "550",
          "usage_limits": {
            "rolling_24h": { "amount_usd_gt": "600", "tx_count_gt": 5 }
          }
        }
      }
    }
  ],
  "duration_seconds": 7776000,
  "completion_conditions": [
    { "type": "tx_count", "threshold": "12" }
  ]
}
```

## State machine

A pact moves through the following states after submission:

```
PENDING_APPROVAL → ACTIVE → COMPLETED
                          → EXPIRED
                          → REVOKED
              → REJECTED
```

| State | What it means |
|---|---|
| `PENDING_APPROVAL` | Agent submitted the pact. Waiting for owner approval in the Human App. |
| `ACTIVE` | Owner approved. The agent now has a scoped delegation and pact-scoped API key to operate within the pact's boundaries. |
| `REJECTED` | Owner rejected the pact. No access was granted. |
| `COMPLETED` | A completion condition was met (e.g., 12 transactions completed). Access revoked automatically. |
| `EXPIRED` | The pact's duration elapsed. Access revoked automatically. |
| `REVOKED` | Owner revoked the pact manually. Access revoked immediately. |

When a pact reaches any terminal state (`COMPLETED`, `EXPIRED`, `REVOKED`, `REJECTED`), the delegation and pact-scoped API key are both revoked. The agent can no longer transact.

## Pact, delegation, and policy relationship

When the owner approves a pact, Cobo Agentic Wallet creates the underlying infrastructure automatically:

```
Pact (what the owner approves)
  │
  ├── Delegation (scoped access grant)
  │     └── derived scope, wallet, operator, expiry
  │
  ├── Inline policies (spending rules)
  │     └── transfer limits, contract allowlists, rolling budgets
  │
  └── Pact-scoped API key (operator credential)
        └── bound to the delegation, returned only to the agent
```

You never need to call `caw delegation create` or `caw policy create` when using pacts — the pact approval flow handles both automatically.

## Submit a pact

<Tabs>
  <Tab title="CLI">
```bash
# Submit with a PactSpec JSON file
caw pact submit \
  --wallet-id <WALLET_UUID> \
  --intent "DCA $500/week into ETH on Base for 3 months" \
  --spec-file pact-spec.json \
  --wait

# Submit with inline options (simplified)
caw pact submit \
  --wallet-id <WALLET_UUID> \
  --intent "Transfer small amounts for testing" \
  --max-tx 100 \
  --duration 86400 \
  --wait

# The --wait flag blocks until the owner approves or rejects.
# Without --wait, the command returns the pact ID immediately.
```

## Poll for activation

After submitting, poll until the pact becomes `active`. When active, the response includes the pact-scoped API key your agent uses for subsequent operations.

<Tabs>
  <Tab title="CLI">
```bash
# Poll status (triggers lazy activation)
caw pact status <PACT_ID>

# List all pacts
caw pact list --format table

# List active pacts only
caw pact list --status active --format table

# Show full detail
caw pact show <PACT_ID>
```
  </Tab>
  <Tab title="Python SDK">
```python
import asyncio

async def wait_for_activation(client, pact_id: str):
    while True:
        pact = await client.get_pact(pact_id)
        if pact["status"] == "active":
            # pact["api_key"] contains the pact-scoped key
            return pact
        if pact["status"] in ("rejected", "expired", "revoked"):
            raise RuntimeError(f"Pact {pact_id} ended in state: {pact['status']}")
        await asyncio.sleep(10)
```
  </Tab>
  <Tab title="TypeScript SDK">
```typescript
async function waitForActivation(client: WalletAPIClient, pactId: string) {
  while (true) {
    const pact = await client.getPact(pactId);
    if (pact.status === 'active') return pact;
    if (['rejected', 'expired', 'revoked'].includes(pact.status)) {
      throw new Error(`Pact ${pactId} ended in state: ${pact.status}`);
    }
    await new Promise(resolve => setTimeout(resolve, 10000));
  }
}
```

</Tabs>

## Revoke a pact

Only the wallet owner can revoke an active pact. Revoking immediately removes the delegation and invalidates the pact-scoped API key.

<Tabs>
  <Tab title="CLI">
```bash
caw pact revoke <PACT_ID>
```
  </Tab>
  <Tab title="Python SDK">
```python
await client.revoke_pact(pact_id)
```
  </Tab>
  <Tab title="TypeScript SDK">
```typescript
await client.revokePact(pactId);
```

## View pact events

Every pact records lifecycle events you can query for auditing or debugging.

<Tabs>
  <Tab title="CLI">
```bash
caw pact events <PACT_ID> --format table
```
  </Tab>
  <Tab title="Python SDK">
```python
events = await client.list_pact_events(pact_id)
```
  </Tab>
  <Tab title="TypeScript SDK">
```typescript
const events = await client.listPactEvents(pactId);
```
  </Tab>
</Tabs>

| Event | What it records |
|---|---|
| `submitted` | Agent submitted the pact for approval |
| `activated` | Owner approved; delegation and API key created |
| `rejected` | Owner rejected the pact |
| `completed` | Completion condition was met |
| `expired` | Duration elapsed |
| `revoked` | Owner revoked an active pact |

## Webhook events

Cobo Agentic Wallet fires webhook events at each stage of the pact lifecycle.

| Event | When |
|---|---|
| `pact_approval.requested` | Pact submitted, awaiting owner approval |
| `pact.activated` | Owner approved, delegation created |
| `pact.rejected` | Owner rejected |
| `pact.expired` | Approval timed out or duration elapsed |
| `pact.revoked` | Owner revoked an active pact |
| `pact.completed` | Completion condition met |
