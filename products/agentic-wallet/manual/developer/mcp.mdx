Use this path when your runtime already supports MCP (Claude Desktop, Claude Code, Cursor, and similar clients).

## 5-minute outcome

1. Start the Cobo MCP stdio server
2. Submit a pact and wait for owner activation notification
3. Invoke wallet tools from your MCP client
4. Observe policy denial returned as tool content
5. Retry with compliant params from `suggestion`
6. Confirm via audit logs

## Step 1: Install MCP extra

```bash
pip install "cobo-agent-wallet[mcp]"
```

## Step 2: Configure API access

```bash
export AGENT_WALLET_API_URL=https://api.sandbox.agenticwallet.cobo.com
export AGENT_WALLET_API_KEY=your-api-key
```

## Step 3: Start stdio MCP server

```bash
python -m cobo_agentic_wallet.mcp
```

## Step 4: Register server in MCP client

Example (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "cobo-agent-wallet": {
      "command": "python",
      "args": ["-m", "cobo_agentic_wallet.mcp"],
      "env": {
        "AGENT_WALLET_API_URL": "https://api.sandbox.agenticwallet.cobo.com",
        "AGENT_WALLET_API_KEY": "your-api-key"
      }
    }
  }
}
```

## Step 5: Run denial plus correction sequence

In your MCP client, execute this flow:

1. Submit a pact for approval: use `transfer_tokens` tool description to explain to the agent what pact is needed, or submit the pact via the Python SDK (`client.submit_pact(...)`) before starting the MCP session. Wait for the owner to approve in the Human App before proceeding.
2. `list_wallets`
3. `transfer_tokens` with compliant amount (success)
4. `transfer_tokens` with too-large amount (policy denial)
5. Parse `suggestion` from tool output and retry with smaller amount
6. `get_audit_logs` to verify allowed plus denied events

Policy denials are intentionally returned as normal tool content (not protocol-level tool failures), so the LLM can reason and retry.

## Quick reference tool set

- `list_wallets`
- `get_balance`
- `transfer_tokens`
- `list_transactions`
- `get_audit_logs`
- `create_delegation`

## Go further

The MCP server exposes the standard wallet tool set. You can extend it with custom tools and other integration paths:

- **Add custom MCP tools** — define additional MCP tools alongside Cobo's for your own business logic (e.g. a `check_price` tool that fetches a token price before submitting a transfer).
- **Use the Python SDK for non-MCP operations** — pact lifecycle management, policy configuration, and audit queries are all available via `WalletAPIClient` for operations you want to handle outside the LLM loop.
- **Use framework integrations for code-based agents** — if you're building a Python agent with LangChain, OpenAI Agents SDK, Agno, or CrewAI, use the framework-native toolkit instead of the MCP server. You get the same tools natively integrated with your agent's tool-calling mechanism.
