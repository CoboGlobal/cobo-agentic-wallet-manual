Cobo Agentic Wallet supports three integration modes, each defining a different split between what you build and what Cobo provides. All modes give your agent pact-based authorization — structured, enforceable delegation agreements with intent, execution plan, permissions, policies, and completion conditions.

## Modes overview

| Mode | You build | Cobo provides | Status |
|---|---|---|---|
| **Agent Mode** | Your CLI, Skills, or agent implementation (using SDK / Toolkit / MCP Server) | Full owner-side stack: Human App, policy management, pact approval | Available now |
| **Core Mode** | Your full custom stack: agent, CLI, Skills, your own owner interface | Core engine API only | |
| **Owner Mode** | Your own owner interface | Agent-side framework | |

## Agent Mode in depth

This is the currently available mode. In Agent Mode:

- **You build**: your agent implementation — using the CLI (`caw`), Skills, SDK, Toolkit, MCP Server, or any framework integration
- **Your users get**: the Human App for pairing agents, reviewing and approving pacts, setting spending rules, and approving transactions — no custom frontend required on your end

Agent Mode is the fastest path to production. Your agent operates through pacts — you ship the agent logic; Cobo ships the owner-side experience.
