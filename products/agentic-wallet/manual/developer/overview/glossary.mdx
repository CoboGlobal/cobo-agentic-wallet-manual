## Product

| Term | Definition |
|---|---|
| **Cobo Agentic Wallet** | The product name. A wallet infrastructure provided by Cobo for individual users, enabling AI agents to operate wallets on their behalf. |
| **Cobo WaaS** | Cobo Wallet-as-a-Service. The underlying blockchain infrastructure platform that Cobo Agentic Wallet is built on. |
| **Human App** | The mobile application through which human users manage their wallets, MPC key shares, and signing operations. |

## Identity & Authorization

| Term | Definition |
|---|---|
| **Principal** | Any entity that can own wallets and interact with the system. Can be of type `human` or `agent`. |
| **Owner** | A principal who owns a wallet. Has full control including policy management (`can_manage`, `can_set_policy`). |
| **Operator** | A principal who has been delegated scoped access to a wallet by its owner. Cannot modify policies or manage the wallet itself. |
| **Delegation** | A grant from an owner to an operator, scoped to a specific wallet, with defined permissions and optional policies. |
| **API Key** | A credential bound to a principal, used to authenticate requests via the `X-API-Key` header. Has scopes and an optional expiry. |
| **Pact** | A task-level authorization request from an agent to its owner. A pact specifies what the agent wants to do (intent), what permissions it needs, spending constraints, and duration. When the owner approves, a scoped delegation and pact-scoped API key are created automatically. The agent operates within those constraints until the pact expires, completes, or is revoked. |
| **Pact-scoped API Key** | An API key created when a pact is activated. It is bound to the delegation created by the pact and is returned to the agent. When the pact terminates for any reason, this key is immediately revoked. |

## Policy & Operations

| Term | Definition |
|---|---|
| **Policy** | A constraint attached to a wallet or delegation that controls what an agent can do. It can include spending limits, transaction frequency caps, chain and token allowlists, address restrictions, and approval thresholds. Policies are evaluated in three tiers — Global → Wallet → Delegation — and the most restrictive tier wins. |
| **Global Policy** | A policy scoped to an owner, applying across all wallets owned by that principal. |
| **Wallet Policy** | A policy scoped to a specific wallet. |
| **Delegation Policy** | A policy scoped to a specific delegation, applying only when that delegated operator acts on the wallet. |
| **Policy Decision** | The outcome of policy evaluation. One of: `ALLOW`, `REQUIRE_APPROVAL`, or `DENY`. |
| **Pending Operation** | A transaction flagged by policy as requiring owner approval before execution. Follows the state machine: `PENDING` → `EXECUTING` → `EXECUTED` / `REJECTED` / `EXPIRED`. |
| **Approval Workflow** | The owner-in-the-loop process for reviewing and approving or rejecting pending operations. |
| **Audit Log** | An immutable record of every operation attempted, including the decision outcome, principal involved, and policy context. |
| **Counter** | A Redis-based sliding window counter that tracks value and count metrics per subject (global / wallet / delegation / principal) to enforce time-window limits. |

## Channels & Onboarding

| Term | Definition |
|---|---|
| **Channel** | A client surface through which a human user interacts with the assistant. |
| **Channel Binding** | The association between a human user's identity and a specific channel account. |
| **Onboarding** | The one-time setup that registers the agent and optionally creates a wallet. Three modes: **supervised** (owner provides pairing code from Web Console, agent creates wallet), **on-device** (owner provides pairing code from Human App, agent registers only, wallet stays on device), **autonomous** (agent self-registers and creates wallet with no pairing code). |
| **Pairing code** | A short-lived token generated by the Web Console or Human App during onboarding. The agent uses it to register with Cobo and get an API key tied to the owner's account. Valid for 15 minutes. |
