## Install

The `caw` CLI is a standalone binary. Download and install it using the bootstrap script:

```bash
curl -sL https://download.agenticwallet.cobo.com/bootstrap-env.sh | bash -s -- --env sandbox
```

This downloads the `caw` binary to `~/.cobo-agentic-wallet/bin/caw`. Add it to your PATH:

```bash
export PATH="$HOME/.cobo-agentic-wallet/bin:$PATH"
```

Verify the installation:

```bash
caw --version
```

## Onboard your agent

Ask your user to generate a pairing code from the Human App (see [Guide your users](/products/agentic-wallet/manual/developer/guide-users) for instructions to share with them). Once you have the code, run:

```bash
caw onboard provision --token <PAIRING_CODE>
```

This creates an API key, binds it to the user's account, and saves config to `~/.cobo-agentic-wallet/config`.

## Verify the setup

```bash
caw onboard self-test
```

## Submit a pact

Before executing transfers, submit a pact to request delegated access from the wallet owner. The owner approves the pact in the Human App.

```bash
# Submit a pact requesting transfer permissions
caw pact submit \
  --intent "Transfer tokens for testing" \
  --execution-plan "Transfer small token amounts for testing purposes." \
  --policies '[{"name":"allow-transfer","type":"transfer","rules":{"effect":"allow","when":{}}}]' \
  --completion-conditions '[{"type":"tx_count","threshold":"100"}]'

# Once active, get the pact ID and check its status:
caw pact status <PACT_ID>
```

<Note>The wallet UUID is resolved from the active wallet profile set during `caw onboard provision`. Required flags are `--intent`, `--execution-plan`, `--policies`, and `--completion-conditions`.</Note>

## Explore your wallet

```bash
# List accessible wallets
caw wallet list

# Check balance
caw wallet balance

# Get wallet details
caw wallet get
```

## Transfer tokens

```bash
PACT_ID=<pact-id>
DST=0x1111111111111111111111111111111111111111

# Allowed transfer
caw tx transfer "$PACT_ID" --to "$DST" --token-id USDC --amount 2 --chain-id BASE_ETH

# Oversized transfer â€” triggers policy denial
caw tx transfer "$PACT_ID" --to "$DST" --token-id USDC --amount 500 --chain-id BASE_ETH
```

On denial, output includes `code`, `reason`, `details`, and `suggestion`.

## Check audit logs

Audit logs are accessible via the Python SDK. See [Audit and Activity Logs](/products/agentic-wallet/manual/developer/audit) for the full query interface.

```python
from cobo_agentic_wallet import WalletAPIClient
import asyncio

async def main():
    client = WalletAPIClient(base_url=API_URL, api_key=API_KEY)
    logs = await client.list_audit_logs(wallet_id=WALLET_UUID, result="denied")
    for entry in logs.get("items", []):
        print(entry["action"], entry["result"])
    await client.close()

asyncio.run(main())
```

## Go further

The CLI is a complete interface on its own, and it also composes well with other integration paths:

- **Wrap in shell functions** â€” build custom `caw`-based scripts for repeated workflows: nightly balance checks, batched transfers, or policy dry-runs before execution.
- **Call from any language** â€” invoke `caw` via subprocess from Python, Node, Go, or any runtime. The structured JSON output makes it easy to parse results programmatically.
- **Add a skill** â€” install `SKILL.md` so your AI agent understands wallet commands without being explicitly prompted. The skill and CLI work together: the skill provides reasoning context, `caw` provides execution.
- **Upgrade to the Python SDK** â€” for async workflows, pact lifecycle management, or custom error handling with `PolicyDeniedError`, use `WalletAPIClient` directly.
