Use this path when you want the fastest way to get from download to a first working CAW integration.

## Install

The `caw` CLI is a standalone binary. The CLI's environment default is `prod`.

```bash
curl -sL https://download.agenticwallet.cobo.com/bootstrap-env.sh | bash
```

This downloads the `caw` binary to `~/.cobo-agentic-wallet/bin/caw`. Add it to your PATH:

```bash
export PATH="$HOME/.cobo-agentic-wallet/bin:$PATH"
```

Verify the installation:

```bash
caw --version
```

## Pair with the owner wallet

Ask the wallet owner to generate a pairing code from the Cobo Agentic Wallet app. If the owner needs step-by-step pairing instructions, send them to [Connect Agent in Cobo Agentic Wallet App](/products/agentic-wallet/manual/owners/connect-agent-human-app). Once you have the code, run:

```bash
caw onboard provision --token <PAIRING_CODE>
```

This creates an API key, binds it to the owner's account, and saves config to `~/.cobo-agentic-wallet/config`.

## Verify the setup

```bash
caw onboard self-test
```

## First hello-world flow

Before executing blockchain actions, submit a pact to request delegated access from the wallet owner. The owner approves the pact in the Cobo Agentic Wallet app.

```bash
# Submit a pact requesting transfer permissions
caw pact submit \
  --intent "Transfer tokens for testing" \
  --execution-plan "Transfer small token amounts for testing purposes." \
  --policies '[{"name":"allow-transfer","type":"transfer","rules":{"effect":"allow","when":{}}}]' \
  --completion-conditions '[{"type":"tx_count","threshold":"100"}]'

# Once active, get the pact ID and check its status:
caw pact status <PACT_ID>
```

<Note>The wallet UUID is resolved from the active wallet profile set during `caw onboard provision`. Required flags are `--intent`, `--execution-plan`, `--policies`, and `--completion-conditions`.</Note>

## Explore your wallet

```bash
# List accessible wallets
caw wallet list

# Check balance
caw wallet balance

# Get wallet details
caw wallet get
```

## Transfer tokens

```bash
PACT_ID=<pact-id>
DST=0x1111111111111111111111111111111111111111

# Allowed transfer
caw tx transfer "$PACT_ID" --to "$DST" --token-id USDC --amount 2 --chain-id BASE_ETH

# Oversized transfer â€” triggers policy denial
caw tx transfer "$PACT_ID" --to "$DST" --token-id USDC --amount 500 --chain-id BASE_ETH
```

On denial, output includes `code`, `reason`, `details`, and `suggestion`.

## Check audit logs

Audit logs are accessible via the Python SDK. See [Audit and Activity Logs](/products/agentic-wallet/manual/developer/audit) for the full query interface.

```python
from cobo_agentic_wallet import WalletAPIClient
import asyncio

async def main():
    client = WalletAPIClient(base_url=API_URL, api_key=API_KEY)
    logs = await client.list_audit_logs(wallet_id=WALLET_UUID, result="denied")
    for entry in logs.get("items", []):
        print(entry["action"], entry["result"])
    await client.close()

asyncio.run(main())
```

Once the CLI path works, move to an SDK if you want to keep the same flow inside application code:

- **Python SDK** â€” same flow, async client, better for custom logic and long-running runtimes.
- **TypeScript SDK** â€” same flow, typed Node.js client for backend services and agent runtimes.
- **Custom Skill** â€” good when your runtime already supports skill files and you want the lightest path.
