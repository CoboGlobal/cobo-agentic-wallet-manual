This is the canonical developer path for Cobo Agentic Wallet.

## 5-minute outcome

In one run, you will:

1. Onboard and get a wallet (from running `caw onboard provision`)
2. Submit a pact requesting transfer permissions
3. Wait for owner approval
4. Execute one allowed transfer using the pact-scoped API key
5. Trigger one policy denial
6. Self-correct using denial guidance
7. Verify in audit logs

## Prerequisites

- Python 3.11+
- A wallet already provisioned via `caw onboard provision` (see [CLI quickstart](/developer/quickstart/cli))
- Your agent's API key and wallet UUID set as environment variables

## Step 1: Install SDK

```bash
pip install cobo-agent-wallet
```

## Step 2: Set environment variables

```bash
export AGENT_WALLET_API_URL=https://api.sandbox.agenticwallet.cobo.com
export AGENT_WALLET_API_KEY=your-agent-api-key
export AGENT_WALLET_WALLET_ID=your-wallet-uuid
```

## Step 3: Run the end-to-end script

```python title="quickstart.py"
from __future__ import annotations

import asyncio
import os

from cobo_agentic_wallet.client import WalletAPIClient
from cobo_agentic_wallet.errors import PolicyDeniedError

CHAIN_ID = "BASE_ETH"
TOKEN_ID = "USDC"
DESTINATION = "0x1111111111111111111111111111111111111111"


async def main() -> None:
    api_url = os.environ.get("AGENT_WALLET_API_URL", "https://api.sandbox.agenticwallet.cobo.com")
    api_key = os.environ["AGENT_WALLET_API_KEY"]
    wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]

    client = WalletAPIClient(base_url=api_url, api_key=api_key)

    try:
        # Step 1: Submit a pact requesting transfer permissions for 24 hours.
        pact_resp = await client.submit_pact(
            wallet_id=wallet_id,
            intent="Transfer tokens for integration testing",
            spec={
                "permissions": ["write:transfer"],
                "duration_seconds": 86400,
                "policies": [
                    {
                        "name": "max-tx-limit",
                        "type": "transfer",
                        "rules": {
                            "effect": "deny",
                            "deny_if": {"amount_usd_gt": "100"},
                        },
                    }
                ],
            },
        )
        pact_id = pact_resp["pact_id"]
        print(f"PACT SUBMITTED: {pact_id}")

        # Step 2: Poll until the owner approves the pact.
        print("Waiting for owner approval in Web Console or Human App...")
        while True:
            pact = await client.get_pact(pact_id)
            status = pact.get("status", "")
            print(f"  pact status: {status}")
            if status == "active":
                break
            if status in ("rejected", "expired", "revoked", "completed"):
                raise RuntimeError(f"Pact reached terminal status before use: {status}")
            await asyncio.sleep(5)

        print("PACT ACTIVE")

        # Step 3: Use the pact-scoped API key for all subsequent calls.
        pact_api_key = pact["api_key"]
        pact_client = WalletAPIClient(base_url=api_url, api_key=pact_api_key)

        try:
            # Step 4: Execute an allowed transfer (within the 100 USD limit).
            allowed = await pact_client.transfer(
                wallet_id,
                chain_id=CHAIN_ID,
                dst_addr=DESTINATION,
                token_id=TOKEN_ID,
                amount="50",
            )
            print("ALLOWED:", allowed.get("status", "unknown"))

            # Step 5: Trigger a policy denial (amount exceeds 100 USD limit).
            try:
                await pact_client.transfer(
                    wallet_id,
                    chain_id=CHAIN_ID,
                    dst_addr=DESTINATION,
                    token_id=TOKEN_ID,
                    amount="200",
                )
            except PolicyDeniedError as exc:
                suggestion = exc.denial.suggestion or "Retry with amount <= 100"
                retry_amount = str(exc.denial.details.get("limit_value", "100"))
                print("DENIED:", exc.denial.code)
                print("SUGGESTION:", suggestion)

                # Step 6: Self-correct using the denial guidance.
                retried = await pact_client.transfer(
                    wallet_id,
                    chain_id=CHAIN_ID,
                    dst_addr=DESTINATION,
                    token_id=TOKEN_ID,
                    amount=retry_amount,
                )
                print("RETRIED:", retried.get("status", "unknown"))

        finally:
            await pact_client.close()

        # Step 7: Verify allowed and denied events in audit logs.
        logs = await client.list_audit_logs(wallet_id=wallet_id, limit=20)
        items = logs.get("items", []) if isinstance(logs, dict) else []
        allowed_count = sum(1 for item in items if item.get("result") == "allowed")
        denied_count = sum(1 for item in items if item.get("result") == "denied")
        print(f"AUDIT: allowed={allowed_count}, denied={denied_count}")

    finally:
        await client.close()


if __name__ == "__main__":
    asyncio.run(main())
```

## Step 4: Validate output

You should see:

- `PACT SUBMITTED: <pact-id>`
- `PACT ACTIVE`
- `ALLOWED: ...`
- `DENIED: TRANSFER_LIMIT_EXCEEDED` (or equivalent policy code)
- `SUGGESTION: ...`
- `RETRIED: ...`
- `AUDIT: allowed=..., denied=...`

## Go further

`WalletAPIClient` is a direct API wrapper — it gives you full control. You can build on top of it in several ways:

- **Write custom tool functions** — wrap `WalletAPIClient` calls in domain-specific functions that encode your business logic (e.g. `transfer_with_price_check`, `batch_payout`), then expose those as agent tools.
- **Use framework integrations** — `AgentWalletToolkit` exposes the standard wallet tools as native LangChain, OpenAI Agents SDK, Agno, or CrewAI tools. Use it for the standard surface, and add your own custom tools alongside it for extended functionality.
- **Pair with the CLI** — use `caw` for onboarding, debugging, and one-off operations; use `WalletAPIClient` for programmatic agent logic. They share the same config and credentials.
