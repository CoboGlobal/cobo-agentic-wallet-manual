The Skill is a `SKILL.md` file that tells a compatible agent (such as OpenClaw) how to use Cobo Agentic Wallet. Once installed, the agent immediately understands wallet operations, policies, and approvals without any additional configuration.

## Step 1: Install the skill

```bash
npx skills add CoboGlobal/cobo-agentic-wallet-python-sdk --skill cobo-agentic-wallet-dev --yes --global
```

## Step 3: Set up your wallet

Follow the [CLI quickstart](/developer/quickstart/cli) to connect your agent to the Cobo Agentic Wallet service and configure your policies.

## Step 4: Guide your users to onboard

Once the skill is installed, your users need to pair their wallet with your agent. Direct them to generate a pairing code from the Web Console or Human App, then share it with you. You then run:

```bash
caw onboard provision --token <PAIRING_CODE>
```

See [Guide your users](/developer/quickstart/guide-users) for the full onboarding flow to share with your users.

## What the agent can do

Once the skill is installed, your agent can:

- Check wallet balances
- Transfer tokens
- Call smart contracts
- Query transaction history
- Respond correctly to policy denials and approval requests

<Info>No SDK or code changes are needed. The skill works with any agent that supports skill files.</Info>

## Go further

The built-in skill is a starting point. You can adapt and extend it:

- **Fork and customize the skill** — copy `SKILL.md`, rename it, and rewrite the language to match your product's terminology and persona. Your users interact with your brand, not Cobo's.
- **Add custom instructions** — extend the skill with domain-specific guidance: which wallets to use by default, how to handle specific token types, or when to ask the user for confirmation.
- **Combine with CLI** — the skill tells the agent what to do; `caw` executes it. Add custom `caw` shell wrappers for logic the skill doesn't cover, such as batched transfers or conditional policy dry-runs.
- **Upgrade to the Python SDK** — when you need programmatic control, pact lifecycle management, or custom error handling, use `WalletAPIClient` directly alongside or instead of the skill.
