## Batch transfers

When you need to send to multiple recipients, you can submit multiple transfers in parallel. Cobo Agentic Wallet processes them independently — each goes through the policy engine separately.

<Tabs>
<Tab title="CLI">
```bash
# Submit multiple transfers in sequence (shell loop)
WALLET_UUID=<wallet_uuid>
RECIPIENTS=("0xAddr1" "0xAddr2" "0xAddr3")
AMOUNTS=("10" "15" "20")

for i in "${!RECIPIENTS[@]}"; do
  caw tx transfer "$WALLET_UUID" \
    --to "${RECIPIENTS[$i]}" \
    --token-id SETH_USDC \
    --amount "${AMOUNTS[$i]}" \
    --chain SETH \
    --request-id "batch-$(date +%s)-$i"
done
```
</Tab>
<Tab title="Python SDK">
```python
import asyncio
from cobo_agentic_wallet import WalletAPIClient

async def batch_transfer(client: WalletAPIClient, wallet_id: str):
    transfers = [
        {"dst_addr": "0xAddr1...", "amount": "10", "request_id": "batch-001"},
        {"dst_addr": "0xAddr2...", "amount": "15", "request_id": "batch-002"},
        {"dst_addr": "0xAddr3...", "amount": "20", "request_id": "batch-003"},
    ]
    tasks = [
        client.transfer_tokens(
            wallet_uuid=wallet_id,
            chain_id="SETH",
            dst_addr=t["dst_addr"],
            token_id="SETH_USDC",
            amount=t["amount"],
            request_id=t["request_id"],
        )
        for t in transfers
    ]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    for i, result in enumerate(results):
        if isinstance(result, Exception):
            print(f"Transfer {i} failed: {result}")
        else:
            print(f"Transfer {i} status: {result.get('status')}")
```
</Tab>
<Tab title="TypeScript SDK">
```typescript
const transfers = [
  { dst_addr: '0xAddr1...', amount: '10', request_id: 'batch-001' },
  { dst_addr: '0xAddr2...', amount: '15', request_id: 'batch-002' },
  { dst_addr: '0xAddr3...', amount: '20', request_id: 'batch-003' },

  transfers.map(t =>
    client.transferTokens({
      wallet_id: walletId,
      chain_id: 'SETH',
      dst_addr: t.dst_addr,
      token_id: 'SETH_USDC',
      amount: t.amount,

for (const [i, result] of results.entries()) {
  if (result.status === 'rejected') {
    console.log(`Transfer ${i} failed:`, result.reason);
  } else {
    console.log(`Transfer ${i} status:`, result.value.status);
  }
}
```
</Tab>
</Tabs>

<Note>
Each transfer is evaluated independently by the policy engine. A denial on one transfer does not affect the others.
</Note>

## Idempotency with request_id

Pass a unique `request_id` to prevent duplicate transactions. If you resubmit with the same `request_id`, the original transaction is returned without creating a duplicate. Use a stable identifier tied to the business operation (invoice number, order ID, etc.).

```python
# Safe to retry — same request_id returns original result
result = await client.transfer_tokens(
    wallet_uuid=wallet_id,
    chain_id="SETH",
    dst_addr="0xRecipient...",
    token_id="SETH_USDC",
    amount="50",
    request_id="invoice-2026-001",  # same ID = idempotent
)
```

<Tip>
Use a deterministic `request_id` derived from your business logic (e.g., `f"order-{order_id}"`) so retries after network failures are safe without any additional state tracking.
</Tip>

## Speed up a stuck transaction

When a transaction is stuck (gas price too low), you can speed it up by resubmitting to the same nonce with a higher fee (RBF).

<Tabs>
<Tab title="CLI">
```bash
caw tx speedup <WALLET_UUID> <TRANSACTION_UUID> \
  --fee '{"fee_type":"EVM_EIP_1559","max_fee_per_gas":"50000000000","max_priority_fee_per_gas":"2000000000"}'
```
</Tab>
<Tab title="Python SDK">
```python
result = await client.speedup_transaction(
    wallet_uuid=wallet_id,
    transaction_uuid=transaction_uuid,
    fee={
        "fee_type": "EVM_EIP_1559",
        "max_fee_per_gas": "50000000000",
        "max_priority_fee_per_gas": "2000000000",
    },
)
```
</Tab>
<Tab title="TypeScript SDK">
```typescript
const result = await client.speedupTransaction({
  wallet_uuid: walletId,
  transaction_uuid: transactionUuid,
  fee: {
    fee_type: 'EVM_EIP_1559',
    max_fee_per_gas: '50000000000',
    max_priority_fee_per_gas: '2000000000',
  },
});
```
</Tab>
</Tabs>

<Note>
The `--fee` flag is required for speedup. The replacement fee must be higher than the original transaction's fee.
</Note>

## Drop a stuck transaction

Cancel a pending transaction that has not yet confirmed on-chain. For EVM chains, this submits a zero-value replacement transaction to the same nonce with a higher fee.

<Tabs>
<Tab title="CLI">
```bash
caw tx drop <WALLET_UUID> <TRANSACTION_UUID>
```
</Tab>
<Tab title="Python SDK">
```python
await client.drop_transaction(
    wallet_uuid=wallet_id,
    transaction_uuid=transaction_uuid,
)
```
</Tab>
<Tab title="TypeScript SDK">
```typescript
await client.dropTransaction({
  wallet_uuid: walletId,
  transaction_uuid: transactionUuid,
});
```
</Tab>
</Tabs>

## Track transaction status

Poll for transaction completion or subscribe to the SSE event stream for push notifications.

<Tabs>
<Tab title="CLI">
```bash
caw tx get <WALLET_UUID> <TX_ID>
```
</Tab>
<Tab title="Python SDK">
```python
tx = await client.get_transaction_record(
    wallet_uuid=wallet_id,
    record_uuid=tx_id,
)
print(tx["status"])  # pending, broadcasting, confirming, completed, failed
```
</Tab>
<Tab title="TypeScript SDK">
```typescript
const tx = await client.getTransactionRecord({
  wallet_uuid: walletId,
  record_uuid: txId,
});
console.log(tx.status); // pending, broadcasting, confirming, completed, failed
```
</Tab>
</Tabs>

**Transaction status values:**

| Status | Meaning |
|---|---|
| `pending` | Queued, not yet submitted |
| `broadcasting` | Broadcast to the network |
| `confirming` | Waiting for block confirmations |
| `completed` | Confirmed on-chain |
| `failed` | Execution failed |

For real-time push notifications without polling, see [Audit and Activity Logs](/products/agentic-wallet/manual/developer/audit) for the SSE event stream.
