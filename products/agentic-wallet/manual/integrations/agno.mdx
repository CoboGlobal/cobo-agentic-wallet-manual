## Overview

The Agno integration exposes both sync and async tool surfaces and hydrates tool metadata from canonical `ToolDefinition` schemas.

## Installation

<CodeGroup>

```bash pip
pip install "cobo-agent-wallet[agno]"
```

```bash uv
uv sync --extra agno
```

</CodeGroup>

## Quick Start

<Steps>

<Step title="Create the tools">
```python
from cobo_agentic_wallet.client import WalletAPIClient
from cobo_agentic_wallet.integrations.agno import CoboAgentWalletTools

client = WalletAPIClient(base_url="http://localhost:8000", api_key="<api_key>")
tools = CoboAgentWalletTools(client=client)
```
</Step>

<Step title="Use with an Agno agent">
```bash
uv run python examples/agno_agent.py
```
</Step>

</Steps>

## Tool Selection

You can enable or disable specific tools via constructor flags or include/exclude lists.

## Denial Behavior

Policy denials are returned as readable output so agent logic can adjust and retry.

<Accordion title="How denials are surfaced">
  Like other integrations, the Agno adapter returns policy denials as tool output
  text (not exceptions). The denial includes the code, reason, details, and
  suggestion fields that the LLM can parse for self-correction.
</Accordion>
