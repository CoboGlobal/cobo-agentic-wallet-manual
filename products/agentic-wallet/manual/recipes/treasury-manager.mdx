## Business Context

Treasury operations need different risk tiers for different operators. Without policy checks, a junior operator can accidentally submit high-risk contract calls.

Core flow:

- Owner creates low-tier and high-tier delegations
- Owner adds contract/function allowlist policy
- Operators evaluate allowed and blocked paths
- Blocked path returns denial context
- Operator retries with approved contract/function and confirms via audit logs

<Accordion title="Policy Architecture">
```text
Owner (human)
  -> Low operator delegation: max_per_tx=25, daily=200, monthly=1000
  -> High operator delegation: max_per_tx=500, daily=5000, monthly=25000
  -> Contract policy:
       - effect: allow
       - contract allowlist + function allowlist
```
</Accordion>

## Success + Denial + Retry Flow

```bash
WALLET_UUID=<wallet_uuid>
TREASURY_CONTRACT=<allowlisted_treasury_contract>

# Allowed call — approved contract and function
caw tx call "$WALLET_UUID" --contract "$TREASURY_CONTRACT" --calldata <allowed_function_calldata> --chain ETH

# Blocked call — function not in allowlist
caw tx call "$WALLET_UUID" --contract "$TREASURY_CONTRACT" --calldata <blocked_function_calldata> --chain ETH

# Retry with approved function from denial suggestion
caw tx call "$WALLET_UUID" --contract "$TREASURY_CONTRACT" --calldata <allowed_function_calldata> --chain ETH
```

## Audit Trail

```bash
caw audit logs --wallet "$WALLET_UUID" --action contract_call --limit 20
```
