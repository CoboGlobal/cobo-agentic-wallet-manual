## Business Context

An agent monitors subscription expiry dates for a company's SaaS tools, auto-renews them before they expire, and alerts the owner when the wallet balance drops below a threshold. The pact restricts transfers to a fixed allowlist of provider payment addresses, caps each renewal at $500, and enforces a $2,000 monthly spending ceiling.

<Accordion title="Permission Architecture">
```text
Owner (human)
  -> Pact (1 year)
       - Operator: agent principal
       - Permissions: [write:transfer, read:wallet]
       - Policies:
           - allowed_destinations: [GITHUB, LINEAR, FIGMA payment addresses]
           - max_per_tx_usd: 500
           - rolling_720h_usd: 2000
```
</Accordion>

## Pact Specification

```json
{
  "permissions": ["write:transfer", "read:wallet"],
  "policies": [
    {
      "name": "allow-subscriptions",
      "type": "transfer",
      "rules": {
        "effect": "allow",
        "when": {
          "dst_addr_in": [
            "<GITHUB_PAYMENT_ADDRESS>",
            "<LINEAR_PAYMENT_ADDRESS>",
            "<FIGMA_PAYMENT_ADDRESS>"
          ]
        },
        "deny_if": {
          "amount_usd_gt": "500",
          "usage_limits": {
            "rolling_720h": { "amount_usd_gt": "2000" }
          }
        }
      }
    }
  ],
  "duration_seconds": 31536000
}
```

### Step 3: Owner approves  in Human App

The agent notifies the owner that an pact is pending. The owner reviews the destination address allowlist, the $500 per-renewal cap, the $2,000 rolling 30-day budget, and the 1-year duration, then approves.

### Step 4: Agent monitors and renews

The agent runs a daily check. It verifies the wallet balance, checks each subscription's expiry, and transfers USDC to renew any subscription due within the configured lead time.

async function checkAndRenew(pact: { api_key: string }) {

  const pactClient = new WalletAPIClient({ apiKey: pact.api_key });

    const usdcBalance = balances.items?.find(b => b.token_id === 'SETH_USDC')?.amount ?? '0';

<Warning>
The `request_id` field ensures idempotency — if the renewal is retried (e.g., after a network failure), the same `request_id` prevents a duplicate transfer. Use a stable identifier like `renewal-{service}-{year-month}`.
</Warning>
