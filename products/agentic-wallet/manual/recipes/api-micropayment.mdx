## Business Context

Agents paying for data or compute APIs should handle tiny recurring payments without exposing a wallet to runaway spend.

<Accordion title="Policy Architecture">
```
Owner (human)
  -> Delegation (operator: API agent)
       - Permissions: [write:transfer]
       - Transfer Policy:
           - max_per_tx_value: 10
           - daily_value_limit: 100
       - Address Policy (allow):
           - data-feed endpoint
           - compute-api endpoint
```
</Accordion>

## Pact Specification

The agent submits this PactSpec when requesting authorization. The owner reviews the endpoint allowlist, per-payment cap, and daily budget before approving.

```json
{
  "permissions": ["write:transfer", "read:wallet"],
  "policies": [
    {
      "name": "allow-api-endpoints",
      "type": "transfer",
      "rules": {
        "effect": "allow",
        "when": {
          "dst_in": [
            "0xDataFeed...",
            "0xComputeAPI..."
          ]
        }
      }
    },
    {
      "name": "deny-over-limits",
      "type": "transfer",
      "rules": {
        "effect": "allow",
        "when": {},
        "deny_if": {
          "amount_usd_gt": "10",
          "usage_limits": {
            "rolling_24h": { "amount_usd_gt": "100" }
          }
        }
      }
    }
  ],
  "duration_seconds": 2592000
}
```

## Full Flow

### Step 1: Owner sets the task

The owner gives the agent a natural-language instruction:

> "Pay for API calls as needed. Max $10 per payment, $100 daily. Only to these endpoints: 0xDataFeed..., 0xComputeAPI.... Valid 30 days."

### Step 2: Agent submits pact

The agent parses the instruction and submits a pact for owner approval.

<Tab title="Python SDK">

```python
import asyncio, os
from cobo_agentic_wallet import WalletAPIClient

async def main():
    async with WalletAPIClient(api_key=os.environ["AGENT_WALLET_API_KEY"]) as client:
        wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]

        # Submit pact
        result = await client.submit_pact(
            wallet_id=wallet_id,
            intent=(
                "Pay for API calls: max $10/payment, $100 daily, "
                "approved endpoints only, valid 30 days"
            ),
            spec={
                "permissions": ["write:transfer", "read:wallet"],
                "policies": [
                    {
                        "name": "allow-api-endpoints",
                        "type": "transfer",
                        "rules": {
                            "effect": "allow",
                            "when": {"dst_in": ["0xDataFeed...", "0xComputeAPI..."]},
                        },
                    },
                    {
                        "name": "deny-over-limits",
                        "type": "transfer",
                        "rules": {
                            "effect": "allow",
                            "when": {},
                            "deny_if": {
                                "amount_usd_gt": "10",
                                "usage_limits": {
                                    "rolling_24h": {"amount_usd_gt": "100"}
                                },
                            },
                        },
                    },
                ],
                "duration_seconds": 2592000,
            },
        )
        pact_id = result["pact_id"]
        print(f"Pact {pact_id} submitted — awaiting owner approval in Human App")

        # Poll for activation
        while True:
            pact = await client.get_pact(pact_id)
            if pact["status"] == "active":
                break
            await asyncio.sleep(10)

        print(f"Pact active. Scoped API key: {pact['api_key']}")

asyncio.run(main())
```

import { WalletAPIClient } from 'cobo-agentic-wallet';

const client = new WalletAPIClient({ apiKey: process.env.AGENT_WALLET_API_KEY! });
const walletId = process.env.AGENT_WALLET_WALLET_ID!;

// Submit pact
const result = await client.submitPact({
  wallet_id: walletId,
  intent: 'Pay for API calls: max $10/payment, $100 daily, approved endpoints only, valid 30 days',
  spec: {
    permissions: ['write:transfer', 'read:wallet'],
    policies: [
      {
        name: 'allow-api-endpoints',
        type: 'transfer',
        rules: {
          effect: 'allow',
          when: { dst_in: ['0xDataFeed...', '0xComputeAPI...'] },
        },
      },
      {
        name: 'deny-over-limits',
        type: 'transfer',
        rules: {
          effect: 'deny',
          when: {},
          deny_if: {
            amount_usd_gt: '10',
            usage_limits: {
              rolling_24h: { amount_usd_gt: '100' },
            },
          },
        },
      },
    ],
    duration_seconds: 2592000,
  },
});
console.log(`Pact ${result.pact_id} submitted — awaiting owner approval`);

// Poll for activation
let pact;
do {
  await new Promise(r => setTimeout(r, 10000));
  pact = await client.getPact(result.pact_id);
} while (pact.status === 'pending_approval');

console.log(`Pact active. Scoped API key: ${pact.api_key}`);

### Step 3: Owner approves  in Human App

The pact appears in the Human App with the intent, execution plan, endpoint allowlist, per-payment cap, and daily budget. The owner reviews and approves. The agent receives a pact-scoped API key and can begin paying for API services.

### Step 4: Agent operates

With an active pact, the agent makes micropayments to approved API endpoints within the daily budget.

<Tabs>

<Tab title="CLI">

```bash
WALLET_UUID=<wallet_uuid>
DATA_FEED=0xDataFeed...
COMPUTE_API=0xComputeAPI...

# Allowed payment — within per-tx and daily limits
caw tx transfer "$PACT_ID" --to "$DATA_FEED" --token-id SETH_USDC --amount 5 --chain-id SETH

# Oversized payment — exceeds $10 per-tx cap, triggers policy denial
caw tx transfer "$PACT_ID" --to "$COMPUTE_API" --token-id SETH_USDC --amount 150 --chain-id SETH

# Retry with compliant amount
caw tx transfer "$PACT_ID" --to "$COMPUTE_API" --token-id SETH_USDC --amount 10 --chain-id SETH
```
