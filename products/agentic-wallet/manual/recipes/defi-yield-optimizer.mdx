## Business Context

Yield agents can rebalance and harvest continuously, but every route and parameter envelope must stay inside owner-approved policy boundaries.

<Accordion title="Contract-Call Guardrail Architecture">
```text
Owner (human)
  -> Optimizer Delegation (agent)
       - write:contract_call
       - allowed_contracts: [Aave Pool, Uniswap Router]
       - allowed_functions: [deposit(), swap()]
       - parameter_constraints:
           - slippage_bps <= 50
           - min_out >= 100
```
</Accordion>

## Full Flow

### Step 1: Owner sets the task

The owner gives the agent a natural-language instruction:

> "Run my yield optimizer. Only Aave and Uniswap on BASE_ETH. Max slippage 50 bps. Valid 30 days."

### Step 2: Agent submits pact

The agent parses the instruction and submits a pact for owner approval.

### Step 3: Owner approves  in Human App

The pact appears in the Human App with the intent, execution plan, contract allowlist, allowed functions, and slippage constraint. The owner reviews and approves. The agent receives a pact-scoped API key and can begin operating.

### Step 4: Agent operates

With an active pact, the agent rebalances positions within the approved protocol boundaries. ERC-20 interactions follow the two-step approve-then-call pattern: first approve the contract to spend the token, then call the deposit or swap function.

<Tabs>

<Tab title="CLI">

```bash
WALLET_UUID=<wallet_uuid>
AAVE_POOL=<aave_pool_contract>
UNISWAP_ROUTER=<uniswap_router_contract>
USDC=<usdc_contract_address>

# Step 1: Approve Aave Pool to spend USDC (ERC-20 approve)
caw tx call "$WALLET_UUID" \
  --contract "$USDC" \
  --calldata <approve_aave_calldata> \
  --chain BASE_ETH

# Step 2: Deposit USDC into Aave — allowed contract, compliant slippage
caw tx call "$WALLET_UUID" \
  --contract "$AAVE_POOL" \
  --calldata <deposit_calldata_slippage_30bps> \
  --chain BASE_ETH

# Blocked call — slippage exceeds 50 bps policy limit
caw tx call "$WALLET_UUID" \
  --contract "$AAVE_POOL" \
  --calldata <deposit_calldata_slippage_100bps> \
  --chain BASE_ETH

# Retry with compliant slippage
caw tx call "$WALLET_UUID" \
  --contract "$AAVE_POOL" \
  --calldata <deposit_calldata_slippage_30bps> \
  --chain BASE_ETH
```

```python
import asyncio, os
from cobo_agentic_wallet import WalletAPIClient

async def main():
    # Use the pact-scoped API key returned after approval
    async with WalletAPIClient(api_key=os.environ["PACT_API_KEY"]) as pact_client:
        wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]
        aave_pool = "0xAavePool..."
        usdc = "0xUSDC..."

        # Step 1: Approve Aave Pool to spend USDC (ERC-20 approve)
        approve_calldata = "0x095ea7b3"  # approve(address,uint256) encoded
        await pact_client.call_contract(
            wallet_id=wallet_id,
            chain_id="BASE_ETH",
            contract_addr=usdc,
            calldata=approve_calldata,
        )

        # Step 2: Deposit USDC into Aave — compliant slippage
        deposit_calldata = "0xe8eda9df"  # deposit(...) encoded with slippage_bps=30
        result = await pact_client.call_contract(
            wallet_id=wallet_id,
            chain_id="BASE_ETH",
            contract_addr=aave_pool,
            calldata=deposit_calldata,
        )
        print(f"Deposit submitted: {result['tx_id']}")

asyncio.run(main())
```
