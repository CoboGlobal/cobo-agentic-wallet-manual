## Business Context

Autonomous commerce between agents needs two controls at once: settlement should happen only after delivery, and every payout must stay inside hard policy limits.

<Accordion title="Agent Owner + Delivery-Gated Settlement Architecture">
```text
Buyer Owner (principal_type="agent")
  -> Delegates to Buyer Execution Agent
       - write:transfer
       - max_per_tx_value: 5
       - daily_value_limit: 20
       - address allowlist: seller settlement address

Settlement flow:
  order pending -> delivery not verified -> no payment
  delivery verified -> attempt settlement -> deny/retry if over cap
```
</Accordion>

## Pact Specification

The buyer agent submits this PactSpec when requesting settlement authorization. The owner reviews the per-payout cap, daily limit, and seller address allowlist before approving.

```json
{
  "permissions": ["write:transfer", "read:wallet"],
  "policies": [
    {
      "name": "allow-seller-only",
      "type": "transfer",
      "rules": {
        "effect": "allow",
        "when": {
          "dst_in": ["0xSeller..."]
        }
      }
    },
    {
      "name": "deny-over-settlement-limits",
      "type": "transfer",
      "rules": {
        "effect": "deny",
        "when": {},
        "deny_if": {
          "amount_usd_gt": "5",
          "usage_limits": {
            "rolling_24h": { "amount_usd_gt": "20" }
          }
        }
      }
    }
  ],
  "duration_seconds": 2592000
}
```

## Full Flow

### Step 1: Owner sets the task

The buyer owner gives the agent a natural-language instruction:

> "Run settlement for agent commerce. Pay the seller up to $5 per delivery, max $20/day. Only to seller address 0xSeller.... Valid 30 days."

### Step 2: Agent submits pact

The buyer agent parses the instruction and submits a pact for owner approval.

import asyncio, os
from cobo_agentic_wallet import WalletAPIClient

async def main():
    async with WalletAPIClient(api_key=os.environ["AGENT_WALLET_API_KEY"]) as client:
        wallet_id = os.environ["AGENT_WALLET_WALLET_ID"]

        # Submit pact
        result = await client.submit_pact(
            wallet_id=wallet_id,
            intent=(
                "Run agent settlement: max $5/delivery, $20 daily, "
                "seller address only, valid 30 days"
            ),
            spec={
                "permissions": ["write:transfer", "read:wallet"],
                "policies": [
                    {
                        "name": "allow-seller-only",
                        "type": "transfer",
                        "rules": {
                            "effect": "allow",
                            "when": {"dst_in": ["0xSeller..."]},
                        },
                    },
                    {
                        "name": "deny-over-settlement-limits",
                        "type": "transfer",
                        "rules": {
                            "effect": "deny",
                            "when": {},
                            "deny_if": {
                                "amount_usd_gt": "5",
                                "usage_limits": {
                                    "rolling_24h": {"amount_usd_gt": "20"}
                                },
                            },
                        },
                    },
                ],
                "duration_seconds": 2592000,
            },
        )
        pact_id = result["pact_id"]
        print(f"Pact {pact_id} submitted — awaiting owner approval in Human App")

        # Poll for activation
        while True:
            pact = await client.get_pact(pact_id)
            if pact["status"] == "active":
                break
            await asyncio.sleep(10)

        print(f"Pact active. Scoped API key: {pact['api_key']}")

asyncio.run(main())

### Step 3: Owner approves  in Human App

The pact appears in the Human App with the intent, execution plan, seller address allowlist, per-payout cap, and daily limit. The owner reviews and approves. The buyer agent receives a pact-scoped API key and can begin settling deliveries.

### Step 4: Agent operates

With an active pact, the buyer agent settles deliveries. Payments only go to the approved seller address and must stay within per-delivery and daily limits.

<Tab title="CLI">

WALLET_UUID=<wallet_uuid>

# Delivery verified — allowed settlement within payout cap
caw tx transfer "$WALLET_UUID" --to "$SELLER" --token SETH_USDC --amount 4 --chain SETH

# Over-cap payout — exceeds $5 per-tx limit, triggers policy denial
caw tx transfer "$WALLET_UUID" --to "$SELLER" --token SETH_USDC --amount 50 --chain SETH

# Retry with compliant amount
caw tx transfer "$WALLET_UUID" --to "$SELLER" --token SETH_USDC --amount 5 --chain SETH

async def verify_delivery(order_id: str) -> bool:
