## Business Context

Cross-chain agents can move fast and lose faster if chains, routers, and slippage are unconstrained. You need policy checks that block unsafe paths and return enough context for automatic retries.

Core flow:

- Owner defines chain-level and function-level allowlists
- Policy enforces slippage and minimum output constraints
- Agent evaluates high-slippage route (deny)
- Agent retries with compliant slippage (allow)
- Agent verifies outcomes through audit logs

<Accordion title="Policy Architecture">
```text
Owner (human)
  -> Policy
       - allowed_chains: [eth, bsc_bnb]
       - router allowlist per chain
       - function allowlist per chain
       - max_slippage_bps: 50
       - min_output_amount: 100
```
</Accordion>

## Success + Denial + Retry Flow

```bash
WALLET_UUID=<wallet_uuid>
CONTRACT=<allowlisted_router_contract>

# Allowed call — compliant slippage within policy bounds
caw tx call "$WALLET_UUID" --contract "$CONTRACT" --calldata <calldata_slippage_30bps> --chain ETH

# Blocked call — slippage exceeds policy limit
caw tx call "$WALLET_UUID" --contract "$CONTRACT" --calldata <calldata_slippage_120bps> --chain ETH

# Retry with compliant slippage from denial suggestion
caw tx call "$WALLET_UUID" --contract "$CONTRACT" --calldata <calldata_slippage_30bps> --chain ETH
```

## Audit Trail

```bash
caw audit logs --wallet "$WALLET_UUID" --action contract_call --limit 20
```
