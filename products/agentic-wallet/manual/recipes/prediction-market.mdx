## Business Context

An agent monitors prediction market events on Polymarket, places conditional bets based on external data sources, and automatically claims winnings when events resolve. Without enforced spending limits, a single runaway bet could exhaust the wallet. A pact scopes the agent to a specific contract, caps each bet at $100, and auto-expires after 7 days.

<Accordion title="Permission Architecture">
```text
Owner (human)
  -> Pact (7 days)
       - Operator: agent principal
       - Permissions: [write:contract_call, read:wallet]
       - Policies:
           - allowed_contracts: [POLYMARKET_CTF_EXCHANGE]
           - chain: BASE_ETH
           - max_per_operation_usd: 100
```
</Accordion>

## Pact Specification

```json
{
  "permissions": ["write:contract_call", "read:wallet"],
  "policies": [
    {
      "name": "allow-polymarket",
      "type": "contract_call",
      "rules": {
        "effect": "allow",
        "when": {
          "chain_in": ["BASE_ETH"],
          "target_in": [
            {
              "chain_id": "BASE_ETH",
              "contract_addr": "<POLYMARKET_CTF_EXCHANGE>"
            }
          ]
        },
        "deny_if": {
          "amount_usd_gt": "100"
        }
      }
    }
  ],
  "duration_seconds": 604800
}
```

## Step-by-Step

### Step 1: Owner sets the task

> **Owner → Agent:** "Monitor the presidential election event on Polymarket. Bet on the outcome if odds are above 60%, max $100 per position, for the next 7 days."

### Step 2: Agent submits pact

The agent constructs a pact from the owner's instruction and submits it for approval.

### Step 3: Owner approves  in Human App

The agent notifies the owner that an pact is pending  in the Human App. The owner reviews the intent, the allowed contract (`<POLYMARKET_CTF_EXCHANGE>` on Base), and the $100 per-bet spending cap, then approves. The agent receives a pact-scoped API key to begin operating.

<Note>
The owner can revise the proposed permissions or spending limits before approving. The agent will operate under the owner's revised terms.
</Note>

### Step 4: Agent monitors and places bets

After the pact is active, the agent polls for favorable odds and places bets through the Polymarket CTF Exchange. Buying a position requires two on-chain operations: approve USDC spending, then call the exchange.

<Tabs>
  <Tab title="CLI">

if (( $(echo "$ODDS > 0.60" | bc -l) )); then
  # Step 1: Approve USDC for the CTF Exchange

    --calldata "$(cast calldata 'approve(address,uint256)' '<POLYMARKET_CTF_EXCHANGE>' 100000000)"

    --to "<POLYMARKET_CTF_EXCHANGE>" \
    --calldata "$(cast calldata 'buyConditionalTokens(bytes32,uint256,uint256)' '<CONDITION_ID>' 100000000 0)"
fi

import { WalletAPIClient } from 'cobo-agentic-wallet';
import { encodeFunctionData, erc20Abi } from 'viem';

const CTF_EXCHANGE_ABI = [...]; // Polymarket CTF Exchange ABI

async function monitorAndBet(pact: { api_key: string }) {
  const walletId = process.env.AGENT_WALLET_WALLET_ID!;
  const pactClient = new WalletAPIClient({ apiKey: pact.api_key });

  while (true) {
    const odds = await fetchOdds('<EVENT_ID>'); // placeholder

    if (odds > 0.60) {
      const amountUsdc = 100n * 10n ** 6n; // $100 USDC

      // Step 1: Approve USDC
      const approveCalldata = encodeFunctionData({
        abi: erc20Abi,
        functionName: 'approve',
        args: ['<POLYMARKET_CTF_EXCHANGE>', amountUsdc],
      });
      await pactClient.contractCall({
        wallet_id: walletId,
        chain_id: 'BASE_ETH',
        contract_addr: '<USDC_ADDRESS>',
        calldata: approveCalldata,
      });

      // Step 2: Buy position
      const buyCalldata = encodeFunctionData({
        abi: CTF_EXCHANGE_ABI,
        functionName: 'buyConditionalTokens',
        args: ['<CONDITION_ID>', amountUsdc, 0n],
      });
      await pactClient.contractCall({
        wallet_id: walletId,
        chain_id: 'BASE_ETH',
        contract_addr: '<POLYMARKET_CTF_EXCHANGE>',
        calldata: buyCalldata,
      });
      console.log(`Bet placed: $100 USDC on YES at ${(odds * 100).toFixed(0)}% odds`);
    }

    await new Promise(r => setTimeout(r, 300000)); // 5 minutes
  }
}

## Audit

```bash
caw audit logs --wallet "$WALLET_UUID" --action contract_call --limit 20 --format table
```

This returns a table of all contract call operations — each row shows the operation timestamp, contract address, calldata hash, USD value, and policy decision (allow/deny). Use this to verify the agent never exceeded the $100 per-bet cap.
