## Business Context

A capable autonomous agent should both earn revenue and operate expenses, but never run outside owner-defined spending boundaries.

<Accordion title="Dual-Wallet Policy Architecture">
```text
Owner (human)
  -> Income Wallet Delegation (same operator)
       - write:transfer
       - address allowlist: expense wallet address
       - purpose: sweep earned revenue

  -> Expense Wallet Delegation (same operator)
       - write:transfer
       - max_per_tx_value: 75
       - daily_value_limit: 120
       - monthly_value_limit: 500
       - purpose: operational spend under budget
```
</Accordion>

## Success + Denial + Retry Flow

```bash
INCOME_WALLET=<income_wallet_uuid>
EXPENSE_WALLET=<expense_wallet_uuid>
EXPENSE_ADDR=0xExpense000000000000000000000000000000000

# Income sweep — allowed to expense wallet address
caw tx transfer "$INCOME_WALLET" --to "$EXPENSE_ADDR" --token SETH --amount 50 --chain ETH

# Expense transfer — within budget
caw tx transfer "$EXPENSE_WALLET" --to <vendor_addr> --token SETH --amount 50 --chain ETH

# Oversized expense — exceeds per-tx limit, triggers policy denial
caw tx transfer "$EXPENSE_WALLET" --to <vendor_addr> --token SETH --amount 500 --chain ETH

# Retry with compliant amount from denial suggestion
caw tx transfer "$EXPENSE_WALLET" --to <vendor_addr> --token SETH --amount 75 --chain ETH
```

## Audit Trail

```bash
caw audit logs --wallet "$EXPENSE_WALLET" --action transfer --limit 20
```
