## Overview

The MCP server exposes all 6 wallet tools via the Model Context Protocol (MCP) stdio transport, powered by FastMCP. It can be used with Claude Desktop, Claude Code, and other MCP-compatible clients.

## Running the Server

```bash
python -m cobo_agentic_wallet.mcp
```

Or programmatically:

```python
from cobo_agentic_wallet.mcp.server import create_server

server = create_server()
server.run()  # starts stdio transport
```

## Tools

The MCP server exposes 6 tools:

| Tool | Description |
|---|---|
| `list_wallets` | List wallets accessible to the caller. Use this tool to discover wallet UUIDs required by other tools such as get_balance and transfer_tokens |
| `get_balance` | Get wallet balances for a specific wallet |
| `transfer_tokens` | Transfer tokens from a Cobo wallet to a destination address. Organisation spending policies are enforced; the transfer may require approval before it is executed |
| `list_transactions` | List transactions associated with a wallet, optionally filtered by status. Returns both incoming and outgoing transactions in reverse-chronological order |
| `get_audit_logs` | Query the organisation audit log, returning records of actions performed by operators on wallets. Supports filtering by wallet, principal, action type, result, and time range using cursor-based pagination |
| `create_delegation` |  |

## Claude Desktop Configuration

Add this to your Claude Desktop `claude_desktop_config.json`:

<CodeGroup>

```json python (global install)
{
  "mcpServers": {
    "cobo-agent-wallet": {
      "command": "python",
      "args": ["-m", "cobo_agentic_wallet.mcp"],
      "env": {
        "AGENT_WALLET_API_URL": "http://localhost:8000",
        "AGENT_WALLET_API_KEY": "your-api-key"
      }
    }
  }
}
```

```json uv (no global install needed)
{
  "mcpServers": {
    "cobo-agent-wallet": {
      "command": "uv",
      "args": ["run", "--with", "cobo-agent-wallet[mcp]", "python", "-m", "cobo_agentic_wallet.mcp"],
      "env": {
        "AGENT_WALLET_API_URL": "http://localhost:8000",
        "AGENT_WALLET_API_KEY": "your-api-key"
      }
    }
  }
}
```

</CodeGroup>

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `AGENT_WALLET_API_URL` | `'http://localhost:8000'` | API base URL |
| `AGENT_WALLET_API_KEY` | — | API key (required) |
| `AGENT_WALLET_TIMEOUT` | `30.0` | Request timeout in seconds |

## Error Contract

The MCP server handles errors with the following semantics:

- **Policy denials** — Raised as `RuntimeError` with a JSON payload containing `error`, `status_code`, `message`, `code`, `reason`, `details`, and `suggestion`. These become MCP protocol errors (`isError: true`).
- **API errors** — Raised as `RuntimeError` with JSON payload containing `status_code`, `message`, and `response`. These become MCP protocol errors (`isError: true`).
- **Missing API key** — Raises `RuntimeError` at tool invocation time with a descriptive message.

```json
{
  "error": "POLICY_DENIED",
  "code": "TRANSFER_LIMIT_EXCEEDED",
  "reason": "max_per_tx",
  "details": {"limit_value": 100, "requested": 1000},
  "suggestion": "Retry with amount <= 100."
}
```

## Server Configuration

```python
from cobo_agentic_wallet.mcp.server import MCPServerConfig, create_server

config = MCPServerConfig(
    api_url="http://localhost:8000",
    api_key="your-api-key",
    timeout=30.0,
)

server = create_server(config=config)
server.run()
```
