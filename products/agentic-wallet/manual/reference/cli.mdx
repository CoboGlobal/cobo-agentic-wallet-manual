The `caw` CLI lets you manage wallets, addresses, transactions, delegations, policies, and agents from the terminal.

## Installation

```bash
npx skills add CoboGlobal/cobo-agentic-wallet-python-sdk --skill --yes --global
```

## Global options

These options are available on every command.

| Option | Env variable | Default | Description |
|---|---|---|---|
| `--api-key` | `AGENT_WALLET_API_KEY` | â€” | API key for authentication |
| `--api-url` | `AGENT_WALLET_API_URL` | `http://localhost:8000` | API base URL |
| `--timeout` | `AGENT_WALLET_TIMEOUT` | `30.0` | Request timeout in seconds |
| `--format` | â€” | `json` | Output format: `json` or `table` |

## Config file

The CLI persists your API key, API URL, and default wallet UUID in `~/.cobo-agentic-wallet/config` (JSON format). Override the path with the `CAW_CONFIG_PATH` environment variable.

Most commands that accept a wallet UUID as a positional argument will fall back to the `wallet_uuid` stored in the config file if the argument is omitted.

## Output formats

<CodeGroup>

```json JSON (default)
{
  "items": [

  ]
```

```text Table

```

## Error output

Policy denials are written to stderr with structured context and a non-zero exit code:

```json
{"code": "TRANSFER_LIMIT_EXCEEDED", "reason": "max_per_tx", "details": {"limit_value": "100", "requested": "1000"}, "suggestion": "Retry with amount <= 100."}
```

## `caw onboard`

Onboard and verify your agent setup.

### `onboard` (provision)

Provision the agent using a pairing code obtained from the Human App. This creates an API key and binds it to your owner account.

```bash
caw onboard --token caw_xxxxxxxxxxxxxxxx
```

| Option | Required | Description |
|---|---|---|
| `--token` | Yes* | Pairing code from the Human App (`caw_â€¦`) |
| `--invitation-code` | Yes* | Invitation code for autonomous provisioning (alternative to `--token`) |
| `--env` | No | Cobo environment: `sandbox`, `dev`, or `prod` (default: `prod`) |

\* One of `--token` or `--invitation-code` is required.

### `self-test`

Verify that the CLI can authenticate and reach the service, and optionally run a transfer policy smoke test.

```bash
caw onboard self-test
```

### `health`

Check service health without authentication.

```bash
caw onboard health
```

## `caw tx`

Submit and query transactions.

### `transfer`

Transfer tokens with policy enforcement. If a policy denies the transfer, the command exits with a non-zero code and prints the denial details to stderr.

```bash
caw tx transfer [WALLET_UUID] \
  --to 0x1234...abcd \
  --token SETH_USDC \
  --amount 50
```

| Option | Required | Default | Description |
|---|---|---|---|
| `--to` | Yes | â€” | Destination on-chain address |
| `--token` | Yes | â€” | Token ID (e.g. `SETH`, `SETH_USDC`). Use the List supported assets operation to retrieve all supported token IDs |
| `--amount` | Yes | â€” | Transfer amount as a decimal string (e.g. `10.5`) |
| `--chain` | No | `ETH` | Chain ID. Resolved automatically from token ID if omitted |
| `--src-addr` | No | â€” | Source address. Defaults to the wallet's primary address for the chain |
| `--request-id` | No | â€” | Client-supplied idempotency key. If a transfer with the same request-id already exists, the existing record is returned |
| `--fee` | No | â€” | Custom fee parameters as a JSON object (EIP-1559 or SOL). If omitted, the network default is used |
| `--gasless` | No | `false` | Use Cobo Gasless (Fee Station) to cover gas fees |
| `--dry-run` | No | `false` | Simulate: check policy and estimate fee without submitting the transfer |

### `call`

Submit an EVM or Solana contract call with policy enforcement. For EVM chains, provide `--contract` and `--calldata`. For Solana, provide `--instructions`.

```bash
# EVM

  --chain SETH \
  --contract 0xabcd...1234 \
  --calldata 0xa9059cbb...

# Solana
caw tx call [WALLET_UUID] \
  --chain SOL \
  --instructions '[{"programId": "...", "keys": [...], "data": "..."}]'
```

| Option | Required | Default | Description |
|---|---|---|---|
| `--chain` | Yes | `ETH` | Chain ID (e.g. `SETH`, `BASE_ETH`, `SOL`). Use the List supported chains operation to retrieve all supported chain IDs |
| `--contract` | No | â€” | EVM contract address to call |
| `--calldata` | No | â€” | EVM calldata as a hex-encoded string (0x-prefixed) |
| `--value` | No | `0` | Native token amount to send with the call as a decimal string |
| `--src-addr` | No | â€” | Source address. Defaults to the wallet's primary address for the chain |
| `--instructions` | No | â€” | Solana instructions as a JSON array of SolInstruction objects |
| `--address-lookup-tables` | No | â€” | Solana address lookup table accounts as a JSON array |
| `--fee` | No | â€” | Custom fee parameters as a JSON object. If omitted, the network default is used |
| `--request-id` | No | â€” | Client-supplied idempotency key |
| `--gasless` | No | `false` | Use Cobo Gasless (Fee Station) to cover gas fees |
