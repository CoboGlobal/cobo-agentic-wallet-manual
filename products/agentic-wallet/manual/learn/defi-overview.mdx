## What is DeFi

**Decentralized Finance (DeFi)** is a set of financial services built on public blockchains, primarily Ethereum and its compatible chains. Unlike traditional finance, DeFi protocols run as open-source smart contracts that anyone can interact with directly — no accounts, no KYC, no intermediaries.

Everything in DeFi is on-chain: the logic, the balances, and the trade history. Anyone can verify any transaction.

## Key DeFi primitives

| Primitive | What it does | Example protocols |
|---|---|---|
| **Token swaps (DEX)** | Exchange one token for another at a market rate | Uniswap, Curve, PancakeSwap, Orca |
| **Lending and borrowing** | Earn interest by supplying assets; borrow against collateral | Aave, Compound, Morpho |
| **Liquidity provision** | Deposit two tokens into a pool; earn a share of trading fees | Uniswap v3, Curve, Balancer |
| **Liquid staking** | Stake native tokens and receive a liquid receipt token | Lido (stETH), Rocket Pool (rETH), Jito |
| **Yield aggregators** | Auto-route and compound positions across protocols | Yearn, Convex, Beefy |
| **Perpetuals and derivatives** | Leveraged trading and synthetic assets | GMX, dYdX, Synthetix |
| **Bridging** | Move tokens between chains | Across, Stargate, native rollup bridges |

## How DeFi fits into Cobo Agentic Wallet

An AI agent can execute any DeFi operation by calling the relevant smart contract — provided the wallet owner has authorized the contracts and functions via a Pact policy.

The typical agent flow:

1. Owner grants authorization via a Pact — specifying which contracts, functions, and parameter limits are allowed.
2. Agent submits a contract call using `caw tx call` or `client.call_contract()`.
3. Cobo policy engine validates the call against the Pact rules.
4. If compliant: the transaction is signed and broadcast on-chain.
5. If non-compliant: blocked (or queued for manual review if `require_approval` is set).

This model lets agents operate autonomously within defined bounds, with the owner retaining control through policy rather than approving each transaction individually.

## DeFi risks

Agents operating in DeFi must account for these risks:

**Smart contract risk** — Protocols can contain bugs or be exploited. Prefer audited protocols with long track records. Limit exposure to unaudited contracts via Pact contract allowlists.

**Price impact** — On illiquid pairs, a large swap moves the price significantly. Always set `amountOutMinimum` (minimum output) or equivalent slippage protection in swap calls.

**Liquidation risk** — Borrowed positions can be liquidated if collateral value falls below the required health factor. Agents managing leveraged positions must monitor health factor continuously.

**Impermanent loss** — Providing liquidity to a pool may result in less value than simply holding the tokens if prices diverge. Fee income must offset the loss to make LP profitable.

**Oracle manipulation** — Some protocols rely on price oracles that can be manipulated in a single block. Prefer protocols that use time-weighted average prices (TWAP) or multiple oracle sources.

**Bridge risk** — Cross-chain bridges have been among the most exploited contracts in DeFi. Limit bridge exposure via contract allowlists and amount caps in Pact policies.

**Gas spikes** — During network congestion, gas costs can increase dramatically. Cobo Gasless handles this automatically, but agents should still account for the fact that spikes may cause transactions to stall.
