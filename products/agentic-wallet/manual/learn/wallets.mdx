## What is a wallet address

Every blockchain account is identified by an **address** — a public identifier derived from a cryptographic key pair.

- On EVM chains: `0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045` (42 hex characters, `0x`-prefixed)
- On Solana: `vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg` (base58, ~44 characters)

Your address is public and safe to share — it is how others send tokens to you. Think of it like an email address: knowing it gives someone a way to reach you, but not access to your account.

## What is a private key

The **private key** is the secret that proves ownership of an address. It is what you use to sign transactions — authorizing them to execute on-chain. Anyone who knows your private key can sign transactions from that address and move all funds.

In traditional EOA wallets (MetaMask, hardware wallets), the private key is a single 32-byte secret. This creates a single point of failure: if the key is lost, stolen, or compromised, the funds are gone.

## MPC wallets and key shares

Cobo Agentic Wallet uses **multi-party computation (MPC)** to eliminate the single point of failure. Instead of a single private key, the system generates multiple **key shares** distributed across separate parties:

- Agent-managed path: the agent's signing node + Cobo's infrastructure (2-of-2)
- Human App path: your phone's main key + your phone's backup key + Cobo (2-of-3)

Signing a transaction requires at least 2 share holders to cooperate. A single key share is cryptographically useless on its own — it cannot be used to sign anything. Cobo never has enough shares alone to move your funds.

<Info>
The AI agent never has access to private key material. The LLM layer and the signing layer are physically separate. The agent submits requests through the Cobo policy engine, which validates them against your rules before triggering the MPC signing process.
</Info>

## Wallet vs. address

In Cobo Agentic Wallet, a **wallet** is a logical container that can have multiple **addresses** — one per supported chain. Your wallet has a single wallet ID, but distinct on-chain addresses on Ethereum, Base, Polygon, and Solana.

```
Wallet (one wallet ID)
├── 0xAbcDef...  (ETH / BASE_ETH / MATIC — same address across EVM chains)
└── vines1v...   (SOL)
```

EVM chains share a deterministic address derivation, so the same key produces the same address on Ethereum, Base, and Polygon. Solana uses a different key scheme.

## Getting your wallet address

After onboarding, list your wallets and their addresses:

```bash
caw wallet list
```

Or via the Python SDK:

```python
wallets = await client.list_wallets()
for w in wallets["data"]:
    print(w["wallet_id"], w["addresses"])
```

## Key share recovery

If a device holding a key share is lost, key shares can be regenerated through a process called **resharing** — Cobo coordinates this without ever reconstructing the full private key. See [Key share management](/products/agentic-wallet/manual/security/keygen-resharing) for details.
