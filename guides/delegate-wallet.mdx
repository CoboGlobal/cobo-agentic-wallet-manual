title: "Delegate Wallet"
description: "Grant an agent scoped access to a wallet with permissions, policies, and expiry."
---

A **delegation** is how an owner grants a second principal scoped access to one wallet:

```
Owner principal
  └─ creates Delegation
        ├─ operator_id  → which agent can act
        ├─ wallet_id    → on which wallet
        ├─ permissions  → what actions are allowed (e.g. write:transfer)
        ├─ policies     → additional spend limits and approval thresholds
        └─ expires_at   → optional TTL
```

Key constraints:

- One delegation = one wallet + one operator. An operator with access to three wallets needs three delegations.
- A wallet can be delegated to **multiple principals** — each as a separate delegation with its own permissions and policies.
- Operators **cannot re-delegate**. Only the owner can create delegations.
- Owners can **freeze** a delegation at any time, halting all operator actions without revoking it.

## Create a delegation

```python
owner_client = WalletAPIClient(base_url=API_URL, api_key=OWNER_API_KEY)

delegation = await owner_client.create_delegation(
    operator_id=AGENT_PRINCIPAL_ID,
    wallet_id=WALLET_UUID,
    permissions=["write:transfer"],
    policies=[],                              # inline policies (optional)
    expires_at="2026-12-31T23:59:59Z",        # optional TTL
)
print("Delegation ID:", delegation["id"])
```


## Permissions

The `permissions` array controls which operations the agent can request:

| Permission | Allows |
|---|---|
| `read:wallet` | List wallets, get balance, list transactions |
| `write:transfer` | Initiate token transfers |
| `write:contract_call` | Execute smart contract calls |
| `write:manage` | Manage delegations |
| `write:policy` | Manage delegation-scoped policies |
| `operator` | Broad operator — implies `read:wallet` + `write:transfer` + `write:contract_call` |
| `viewer` | **Deprecated.** Alias for `read:wallet` |
| `write:wallet` | **Deprecated.** Alias for `write:transfer` |

**How permissions map to effective access:**

| Effective check | Satisfied by any of |
|---|---|
| `can_view` | `viewer`, `read:wallet`, `operator`, `write:wallet`, `write:transfer` |
| `can_transfer` | `operator`, `write:wallet`, `write:transfer` |
| `can_call_contract` | `operator`, `write:contract_call` |
| `can_set_policy` | `write:policy` |
| `can_manage` | `write:manage` |

<Info>
  `write:contract_call` does **not** imply `can_view`. Add `read:wallet` explicitly if the agent also needs to read the wallet.
</Info>

<Warning>
  `viewer` and `write:wallet` are deprecated. The API will automatically normalize them to `read:wallet` and `write:transfer` respectively. Use the canonical names in new integrations.
</Warning>


## Inline policies

You can attach policies directly on the delegation at creation time:

```python
delegation = await owner_client.create_delegation(
    operator_id=AGENT_PRINCIPAL_ID,
    wallet_id=WALLET_UUID,
    permissions=["write:transfer"],
    policies=[
        {
            "name": "daily-budget",
            "type": "transfer",
            "rules": {
                "effect": "allow",
                "daily_value_limit": "500",
                "max_per_tx_value": "100",
                "approval_threshold_value": "200",  # >200 requires owner approval
            },
        }
    ],
)
```

See [Set Policies](/guides/set-policies) for the full policy schema.


## List delegations

```python
# Owner lists all delegations they created
delegations = await owner_client.list_delegations(limit=50, offset=0)

# Filter by wallet
delegations = await owner_client.list_delegations(wallet_id=WALLET_UUID)
```


## Freeze and unfreeze

Freezing halts all agent operations on the affected delegations **immediately**, without revoking them. Useful for emergency stops.

```python
# Freeze all delegations for this owner
await owner_client.freeze(scope="owner", owner_id=OWNER_ID)

# Freeze all delegations on a specific wallet
await owner_client.freeze(scope="wallet", wallet_id=WALLET_UUID)

# Freeze a specific delegation only
await owner_client.freeze(scope="delegation", delegation_id=DELEGATION_ID)

# Unfreeze (same scope options)
await owner_client.unfreeze(scope="wallet", wallet_id=WALLET_UUID)
```

A frozen delegation returns a 403 with `delegation_frozen` when the agent attempts any operation.


## Revoke a delegation

```python
await owner_client.revoke_delegation(delegation_id=DELEGATION_ID)
```

Revoked delegations are permanently inactive. To restore access, create a new delegation.


## Delegation lifecycle

```
created → active → (frozen ↔ unfrozen) → revoked / expired
```

| Status | Description |
|---|---|
| `active` | Delegation is in effect |
| `expired` | Past `expires_at`, evaluated lazily on next access |
| `revoked` | Permanently deactivated by the owner |
| `frozen` | Temporarily suspended; operations are denied until unfrozen |
